## 批次 1（立即，高 ROI / 低-中成本 / 自包含）

### 类别双真源收敛
- **问题**：`ValidCategories`/`GetSubCategories` 后端硬编码，前端 `categoryOptionsMap`（`AccountManage.vue`）独立再写一份；注释自承"改后端忘改前端→新科目建得了但页签不显示"。真实漂移风险。
- **方案**：新增 `GET /api/finance/accounts/categories` 由后端 `ValidCategories`+`GetSubCategories` 派生返回；前端 `categoryOptionsMap` 改接口取数（保留对老数据"损益"大类名的兼容）。
- **可测**：后端接口返回结构有单测；前端 vite build。

### 科目搜索框 + 全部展开/折叠
- **问题**：扁平自绘树表、无搜索框（后端 `GetSelectorList` 已支持 keyword 但本页未用）、无全部展开/折叠、编码列禁排序 → 大科目表导航成本高。
- **方案**：纯前端——toolbar 加 `a-input` 过滤 `flattenAccounts` 的 code/name，命中把父链 id 灌入 `expandedIds` 自动展开+高亮；加"全部展开/折叠"按钮（展开=遍历有子节点 id；折叠=`new Set()`）。期初表也覆盖搜索。参照 `AuxiliaryPicker.vue` 已有搜索范式。

### 导出 Excel + 导入占位禁用
- **问题**：「更多」菜单导入/导出点了只弹"功能开发中"，与真功能视觉无差别（误导）；导入还逃逸了 SubjectEdit 权限门控。
- **方案**：导出立即用前端 `flattenAccounts` 现有数据生成 Excel（ROI 最高）；导入暂加 `disabled`+tooltip『即将上线』或整组 `v-if=canEdit`。中期再接 `ApplyTemplateAsync` 做真导入。

## 批次 2（中 ROI / 中成本）

### 批量启停 / 批量设辅助核算
- 勾选后唯一批量动作是删除。补『批量启用/停用』（**注意改为显式 set-status 而非翻转**，否则混合勾选各自反向）；补『批量设/清辅助核算』（复用 `update-auxiliary`）。

### 余额方向 switch→radio
- 借/贷是对等枚举不是布尔；改 `a-radio-group`/`a-segmented`『借方/贷方』，与表格/期初表统一。多数处于 disabled、误操作面小，故低 ROI。

### 启停 toggle→setStatus 幂等
- 与刚做的启停校验相关：`toggleAccountStatus` 是 toggle 语义、前端传的 val 被后端忽略、无条件取反，并发下可能落库与意图相反。改 `setStatus(id, enabled)` 绝对赋值。**做这个时复用第 7 轮已加的停用校验逻辑。**

## 批次 3（高 ROI 但大改动，各需独立 spec）

### 辅助核算余额闭环（FinAuxiliaryBalance）
- **真 bug**：`FinAuxiliaryBalance` 表（FAuxiliaryJson 维度+期初/发生/期末六列）**全代码库零写入**（仅 `ReportService.cs:274` 读取一次）→ `AuxiliaryBalanceReport` 整张报表恒空。
- **方案**：复用既有表，分三处补写入：①期初——挂辅助核算末级科目按辅助项下钻录入；②本期——凭证过账按辅助拆分汇总；③结转——期间结转把上期辅助 FEnd 滚到本期 FBegin。**不要**在 FinAccountBalance 加辅助维度字段（已有专表）。

### 结束初始化 / 账套启用闸门
- **缺失的核心会计概念**：`F启用年度`/`F启用期间` 恒 0、无人维护；期初可无限次改、从不冻结；无『结束初始化』闸门。
- **方案**：引入账套初始化状态机（未启用/已启用）：期初页未启用时可编辑→『结束初始化』按钮（强制试算平衡通过后写定启用年度/期间、锁定期初）→已启用改期初须『反初始化』（校验是否已有凭证，跨组织 IgnoreQueryFilters）。可复用 PeriodClosing 已有试算平衡逻辑。**最有价值的中期项**。

## 批次 4（中长期）

- **数量核算闭环**：`FinAccountBalance` 增 FBeginQuantity 等、期初表加数量列、凭证支持数量行、给 FinAccount 增显式『是否数量核算』布尔（现靠 unit 非空隐式判定脆弱）。
- **编码方案配置化**：把 4-2-2-2/4级/99 从三处硬编码（`AccountService` 校验 + `AccountManage` 正则 + 文案）收敛为后端单一常量+接口下发；中期落 `FinAccountSet.FCodeRuleJson`。补企业准则『共同类』(类别6、一级首位放宽到 1-6)。
- **科目角色属性**：FinAccount 增『科目类型(普通/现金/银行/往来)』+『封存』，弹窗暴露；现金流量项目走独立主数据（别塞辅助核算枚举）。需 EF 迁移（MigrationRunner 不可删历史迁移）。
- **科目重分类 / 合并**：改上级整支重编码+冲突校验 / 并转余额+凭证 FAccountId 重指向。**注意排查 `CardFlowVoucherBridge` 等按字符串 code 硬关联科目处**。涉及凭证/余额迁移，单列规划、风险最高。
