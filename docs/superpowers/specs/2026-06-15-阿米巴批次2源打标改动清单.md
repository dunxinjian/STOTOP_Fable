# 阿米巴方案B 批次2「源打标」实施改动清单

## 1. 总览

**一句话目标**：让进入阿米巴报表的每个数据点(billing 聚合 / 自动凭证 / 迁移凭证 / 估值)都按科目契约 `FinAccount.FAuxiliary` 声明的维度，把 `business_direction`(OUT/IN/CMB)、`outlet`、`project`、`department` 等辅助核算**带齐并写进可被 `AuxiliaryMatches` 读到的位置**，使报表能按辅助核算多维过滤；同时新建 `EntryAuxValidator` 把科目契约接到凭证/估值写库口做兜底校验。**本批次只动"机制"，不改 `ClassifyDirection` 语义、不改 `Direction` 字段口径**(留给批次3)。

口径取舍(已在调研中定，本清单遵循)：billing 采用**最小安全改法**——只在 `AuxValues` 写 `business_direction="OUT"`，**不动 `DataPoint.Direction="出港"`**(避免连锁回归 `MapDirectionToBucket`/`ApplyAllocation`/`MapToUnit`)。

## 3. 机制 vs 规则配置(严格区分)

**【机制 — 本批要建/改的代码】**
- A1 billing 写 `AuxValues["business_direction"]="OUT"`；B1 `AuxiliaryMatches` 三个显式 case。
- C1 预加载科目契约字典；C2 ResolveAuxiliary 后的契约补齐回路；C3 `EntryLineV2.DefaultAuxiliaryConfigs` 字段。
- D1 迁移桥归一调用点 + 契约校验接入(两镜像)。
- E1 `EntryAuxValidator` 纯函数；E2 `ValidateEntriesAsync` 挂载 + 严格度开关。
- F1 估值 Direction 回填；F2 估值录入自检告警。
- G1 前端配置字段透传。

**【规则配置 — 业务/记账填的映射值，本批只留接口/配置位，代码不写死】**
- **business_direction 注入值**：出港规则组 fixed=OUT、进港=IN、跨段=CMB —— 填进 C3 的 `DefaultAuxiliaryConfigs`(规则 JSON)，**不在 resolver 写死**。
- **段名→OUT/IN/CMB 归一表**(申通出港费/中转出→OUT、派费/到件→IN、操作无段→CMB；韵达揽→OUT/派→IN；中通发→OUT/派→IN)——若用 dynamic 派生，作为规则配置/可选归一映射填，本批不在代码硬编归一表。
- **裹裹 project**：由哪条凭证生成规则给分录赋 `project=裹裹项目`(fixed)——配置，billing 不加 project。
- **部门→段码**(YWY_OUT/YWY_IN/KF/CZ/YWGL)：在 `department` 辅助核算项里建主数据 + 工资损益项叶 filter 按段分流——主数据/模板配置。
- **预置凭证规则补辅助核算**：`Taicang/init_voucher_rules.py`(:31-102，7 条仅借贷+摘要)需在 `AuxiliaryConfigs`/`DefaultAuxiliaryConfigs` 按科目契约补维度——规则配置，非机制。
- **科目契约本身**：`FinAccount.FAuxiliary` 各科目声明哪些维度(如 500101 含 business_direction，spec:543/新科目表 CSV)——主数据配置，机制只读取。

## 5. 风险与回归点(重点：CardFlow/Express 公共凭证生成路径)

- **R1 — 自动凭证流水线被校验卡死(最高影响面)**：E2 在 `ValidateEntriesAsync` 强校验会让 `CreateAsync` 抛错回滚整组(`AutoVoucherHandler.cs:451-457` 记 failedGroups)。`AutoVoucherAuxiliaryResolver` 未匹配只 warning 不阻断(:92)，故自动凭证天然可能"打标不全"。**必须**：自动凭证期降级 warning+标记、录入/导入期强校验(严格度开关，按调用上下文区分)。
- **R2 — business_direction 取值口径不统一**：billing 的 `dp.Direction` 是中文"出港"，`business_direction` filter 的 Codes 是 OUT。B1 必须 `AuxValues` 在前、`dp.Direction` 仅 fallback；A1 必须确保 billing 写了 `AuxValues=OUT`。否则中文"出港"≠"OUT" 判不命中，billing 点被剔除净利虚高。
- **R3 — 各源 business_direction 必须同口径打标**：只 billing 打 OUT 而 voucher/estimate 不打，会导致出/进港 filter 仅出港命中、进港分录被剔除。本批 A1(billing)+C2(voucher 补齐)+F1(estimate 回填)三源齐改正是为此。
