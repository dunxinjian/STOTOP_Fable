# 外币死字段清理（P2）— 设计

- 日期：2026-06-19
- 范围：删除科目管理页前端的外币死字段（表格列恒空、currency 透传无录入控件）。
- 优先级：P2（消除"有列无值"误导）
- 工作区：worktree `finance-p2b`

## 1. 问题

`AccountManage.vue` 有「外币」表格列、`formData.currency`、`buildPayload` 提交 currency，但弹窗**无任何录入控件**——该列永远空、currency 永远写不进有效值。典型"有列无值"半成品，误导用户以为支持外币（实际单一申通本币，多币种核算整体未闭环）。

## 2. 修复设计（纯前端删除）

删除 `AccountManage.vue` 的 6 处外币死字段：

| 位置 | 内容 |
|---|---|
| 表格 columns | 「外币」列 `{ title: '外币', dataIndex: 'currency', ... }` |
| `formData` | `currency: ''` 初始 |
| `getAccountById` 回填 | `currency: data.currency || ''` |
| `buildPayload` | `currency: formData.currency || null` |
| `resetForm` | `formData.currency = ''` |
| `AccountNode` interface | `foreignCurrency: string`（未使用字段，顺带清理） |

**后端不动**：`FinAccount.FCurrency` / `AccountDto.Currency` 字段保留（不破坏 DB 列与序列化兼容、保留未来多币种余地），仅前端不再暴露。删 `buildPayload` currency 后 `request.Currency=null`，`UpdateAsync` 仍 `FCurrency=request.Currency=null`，而 FCurrency 本就恒 null，无副作用。

## 3. 测试

- 无后端改动 → 无新测试。
- 前端 `npx vite build` 通过 + 手动核对（外币列消失、新增/编辑科目正常保存）。

## 4. 非目标（YAGNI）

- 不删后端 `FCurrency`/`Currency` 字段、DTO。
- 不做多币种核算（外币录入/汇率/原币）。
