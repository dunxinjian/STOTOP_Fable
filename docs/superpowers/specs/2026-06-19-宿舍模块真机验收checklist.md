# 宿舍管理模块 真机验收 Checklist

> 用途：数据库恢复连通后，照此逐项验收宿舍模块（分支 `worktree-dormitory-polish`）。
> 代码层已验证：后端 27 单元测试全绿、vue-tsc 全项目 0 类型错误、vite build / dotnet build 0 错误。本清单覆盖**只有真机能验**的运行时/UX/集成项。

## 0. 环境准备
- [ ] 数据库可达（远程 `47.105.65.51` 恢复，或配置可达的库）。
- [ ] 起后端 `scripts/dev/backend.sh`、前端 `scripts/dev/frontend.sh`（端口 9000/9001）。
- [ ] 确认 baseline 种子已灌（宿舍菜单 FID 700-708 在库；侧栏可见"宿舍管理"）。
- [ ] 切到有宿舍数据的组织（如 org 192）。登录成功、无控制台报错。

## 1. 路由 / 集成（修复重点）
- [ ] 侧栏"宿舍管理"展开 8 子菜单，**逐个点进无 404**（楼栋/入住/费用/设施/报修/访客/卫生/统计）。
- [ ] 楼栋列表"管理床位"→ RoomManage（带 buildingId），**返回**回到楼栋列表（原按 name 跳转必 404，已修）。
- [ ] 切换组织后各列表**自动刷新**为新组织数据。

## 2. 楼栋 → 房间 → 床位
- [ ] 新建楼栋（编码/名称/总楼层/**管理员搜索选择**/类型/地址）；编辑回填管理员**显示姓名**。
- [ ] 楼栋列表显示**房间数/床位数/已占用数**（后端实时统计）。
- [ ] 房间页：新建房间、显示**占用率 occupiedBeds/bedsCount**；总楼层显示正确（原 floorCount 已修）。
- [ ] 管理床位：新建（默认空闲）、编辑可改"维修中"；床位类型显示正确（原漏 BedType 已修）。

## 3. 入住 / 退宿（床位状态联动）
- [ ] 办理入住：**员工搜索选择**、选空闲床位、入住日期 → 床位变"已入住"、占用数 +1。
- [ ] 同一床位**不能重复入住**（已测）。
- [ ] 退宿（弹窗选日期+确认）→ 记录"已退宿"、**床位回"空闲"**、占用数 -1。
- [ ] 列表无幽灵列（已删 employeeCode/部门/预计退宿/实际退宿）。

## 4. 费用 + 分摊（新功能重点）
- [ ] 新建费用：房间/类型/金额/月份/**分摊方式(均摊·固定)**；金额为负被拒；新建默认**"待缴"**（已修）。
- [ ] 列表"分摊方式"列**有值**（原从不发送=空白，已修）。
- [ ] 点"分摊"→ 弹窗显示按在住人分摊：均摊各人=总额/人数且**合计严格等于总额**；固定各人=金额；无在住人提示空。
- [ ] 编辑费用可改缴费状态（待缴/已缴/减免）。

## 5. 设施
- [ ] 选楼栋→房间后新建设施（名称/数量/备注）；列表无幽灵列（已删类型/购买/保修）。
- [ ] 编辑可改**状态**（正常/损坏/报废）且确实保存（原 update 不发 status，已修）。

## 6. 报修工单（契约修复重点）
- [ ] 新建：房间/**报修人搜索选择**/描述/紧急程度 → 列表**描述、紧急程度、报修人都有值**（原全空白，已修）。
- [ ] 按"紧急程度"筛选有效（原字段错名，已修）。
- [ ] "处理"：**处理人搜索选择**/结果/新状态 → **处理人、结果、处理时间都有值**、状态更新（原漏发，已修）。
- [ ] "编辑"（待处理/处理中）可改描述/紧急度（已补）；完成/关闭态按钮置灰。

## 7. 访客（契约修复重点）
- [ ] 新建：房间/姓名/电话/证件/事由/**被访人搜索选择**/到访时间 → 列表**事由、到访时间、被访人都有值**（原全空白，已修）。
- [ ] "登记离开"**有二次确认**，确认后状态"已离开"、离开时间=当前（原 NPE/404，已修）。
- [ ] 编辑可改事由/被访人等。

## 8. 卫生检查
- [ ] 新建：房间/**检查人**/日期/评分(0-100，超范围被拒)/结果；列表检查人有值（原字段错名，已修）。
- [ ] "编辑"可改评分/结果（已补）。

## 9. 统计看板
- [ ] 各指标正确（楼栋/房间/床位/占用率、待处理报修、今日访客、待缴费用）；配色为令牌色。

- [ ] 各页加载态/空态(EmptyState)/错误提示正常；删除均有 popconfirm。
- [ ] 窄屏表格横向滚动正常（responsive 抽查）。
