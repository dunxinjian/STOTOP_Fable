> 注：文件名沿用旧 slug「逐模块巡检」，实际铺开方式已改为**类驱动模板迁移**（见 §4）。

- 日期：2026-06-17
- 定位：**续作**，承接 [`2026-06-16-系统UX-UI统一重设计-design.md`](2026-06-16-系统UX-UI统一重设计-design.md)（阶段0-9 已合并，提交 `a056895`）的 §5.6 残留 + §5.7「逐模块巡检」未竟部分
- 上游契约（不重做、只消费）：`web/docs/TOKENS.md`（令牌真源）、`web/docs/PATTERNS.md`（四类页面骨架 + 共享组件契约）
- 视觉方向：**① 克制收敛**——保留现有 Ant Design + 已落地的「中性墨 + 橙只在交互点缀」观感，只统一秩序，不引入新视觉语言
- 范围：`web/src/views/**` 功能页**消费层**（PC）
- 非范围：移动端（D，上游已对齐令牌，本轮功能页巡检限 PC）

## 0. 为什么是「续作」而不是新方案

多智能体审计（11 agent，跑在 `a056895` 及后续「去橙→中性」精修提交**之后**）确认：

- **地基与外壳已完成**：令牌单一真源、去蓝 221 处、拆后台紫、登录/工作台/全局搜索/顶栏侧栏重做、后端默认色改 `#E85E00`、TOKENS.md / PATTERNS.md 已产出、`config/theme.ts` 已删。
- **`PATTERNS.md` 自己写明「其余进逐模块巡检阶段」**——这步基本没落地。规范建好了却**没人用**，500+ 功能页仍停在旧写法。

因此本 spec 的本质是「**让既有规范真正被采用 + 清掉规范外的存量 + 重做最烂的页**」，**复用优先于新建**，严禁重造已存在的组件与约定。

## 2. 原则

1. **站在 PATTERNS 上**：先复用 `PageHeader` / `StatusTag` / `BaseCard` / `EmptyState` / `.page-toolbar` / `.page-card` / `.page-container` 与 `#toolbar`/`#actions` 约定；确无对应物再新建。
2. **门禁只拦新增，存量按工单分批**：避免一次性卡在 1943 存量上。
3. **逐页深修都基于既有规范**：保证「逐一优化」天然符合统一风格，不越改越花。
4. **样板先行、验证后铺开**：先在 1 个代表页跑通新基建，再横向铺开。

## 3. 阶段 0：新基建 + 样板页（第一可执行批次 · 详写）

### 3.1 复用既有，不新建

| 组件 / 约定 | 状态 | 本轮动作 |
|---|---|---|
| `PageHeader`（4 槽，#toolbar 筛选 / #actions 主操作） | 已建、205 文件用 | 不动；作为筛选条迁移目标 |
| `StatusTag`（type/biz/dot，令牌干净） | 已建、仅 1 处用 | **推广**：替换手写 `.status-tag` 与 `a-tag` 字面色 |
| `BaseCard` / `EmptyState` | 已建 | 推广为默认（表格空态、卡片容器） |
| 全局类 `.page-toolbar(__group/__filters)` / `.page-card` / `.page-section__title` | 已建 | 筛选条 / 卡片 / 区块标题统一指向它们 |

> **筛选条不另建 `FilterBar` 组件**：PATTERNS 已选定「`#toolbar` 槽 + `.page-toolbar__filters` 全局类」方案，再造组件属重复。审计里的 `FilterBar` 提案作废，统一迁到既有标准。

### 3.2 新建组件（3，经用户确认）

- **`DataTable.vue`** —— 包装 `a-table`：统一 `paginationConfig` 默认（`showSizeChanger` / `pageSizeOptions` / `showTotal“共 N 条”`）、`size=small` 密度、内建序号列开关与 index 计算、内建 `EmptyState` 作 `emptyText` 默认、规范 `handleTableChange` 回填、约定 `scroll`；外层可选包 `BaseCard noPadding`。**消除 G6 的 55 文件重复**与各页自写的 `.table-card` `:deep` 覆盖。
- **`StatFilterTabs.vue`** —— 带计数状态快筛条：`tabs=[{key,label,count,color?}]` + `v-model:active`，高亮条走 `--color-primary`、文字 `--text-*`、计数色走令牌（不内联 rgba）。收编 CustomerManage `.status-tab-bar` / ServiceOrderManage `.tab-item` / VehicleManage 顶部 4 张 `a-statistic` KPI 卡三套手写。
- **`PageLayout.vue`** —— 薄基元，根渲染 `div.page-container`，`variant="table"`（默认，保留现有表格 flex 滚动链）/ `"flow"`（纵向多卡片流式，解除 `overflow:hidden` 锁定改 `overflow-y:auto`）。**收编 G9 的 45 处覆写**为一个 prop，消灭 `OrchestrationDetailPage` 那类注释式 hack；与 `PageHeader`（工具栏 Teleport）正交，文档画清「标题→面包屑、操作→#actions、内容→PageLayout」。

> 区块标题**不新建组件**：统一用既有全局类 `.page-section__title` 替换 ContractList 本地 `.section-title`、QuotationEdit `.aside-section-title`（裸 `#262626`/`#f0f0f0`）等；若实测需要「标题 + 右侧操作槽」变体，再随样板页决定是否薄封装 `SectionTitle`（同 FilterBar 的处理原则——复用优先于新建）。

### 3.3 密度全局兜底

`App.vue` 的 `a-config-provider` 增加 `:component-size="'small'"`（或纳入 themeStore），让 `small` 成为真正全局默认；回归验证后**分批删除**页面冗余 `size="small"`。规范：默认即 small，仅需大控件时显式 `middle`/`large`。

### 3.4 新令牌（唯一）

在 `theme.ts` + `variables.scss` 新增 `--avatar-palette-*`（一组头像底色），替代 CustomerManage `bdColors` 裸 hex 数组 + `#d9d9d9` fallback。**其余 `-light` 令牌已存在，禁止新造。**

### 3.5 stylelint 门禁（防回潮）

- husky `pre-commit` + `lint-staged`：**只校验 staged 的 `.vue`/`.scss`**，新增裸 hex 即拦；存量不阻塞。
- CI 增 `lint:style` 步骤。
- 补盲区：`<template>` 内联 style 裸 hex（~245 处）用脚本/规则纳入收敛清单；ECharts 色板 22 文件写入**显式白名单**（canvas 不解析 `var()`，属合理豁免）。
- 存档 baseline 快照：`node_modules/.bin/stylelint "src/views/**/*.vue"` 当前 1943/242，作为递减度量。

### 3.6 样板页 = `/dormitory/buildings`（`BuildingManage.vue`）

审计改判定它为样板（原推荐 `/crm/customers` 本身是重灾区，弃用）：它已用 `PageHeader #toolbar` + `paginationConfig` + `EmptyState` + 标准操作列，唯一硬伤（`#toolbar` 内联 flex / `a-tag` 字面色 / `a-table bordered`）正好验证 **DataTable + StatFilterTabs + StatusTag + #toolbar 迁移** 四件套。备选 `/insurance/policy`（PolicyManage，已用 size=small + #toolbar）。

**产出**：样板跑通后，把「列表页迁移配方」回写进 `PATTERNS.md`（增 DataTable/StatFilterTabs/PageLayout 契约）。

## 4. 阶段 1：类驱动模板迁移（覆盖全部可模板化页面）

铺开方式从「按模块逐个扫」改为「**按页面类逐类定模板、再批量套用**」——同类同配方，一致性与批量效率最高。**特殊页**（工作台/编辑器/多视角报表）不套模板，走阶段2。

### 4.1 页面类与模板（样板验证后批量套）

| 页面类 | 规模 | 模板组成（站在 PATTERNS 上） | 样板页 |
|---|---|---|---|
| 列表页 | ~150+（最大类） | PageLayout + PageHeader(#toolbar 筛选) + **DataTable** + StatusTag + StatFilterTabs(可选) + EmptyState | BuildingManage（= 阶段0 样板，已跑通） |
| 详情页 | ~11 | PageHeader(backTo) + 概览条(StatCard) + 主区/侧栏 或 标签 | TaskDetail |
| 表单/提交页 | ~18 | PageHeader(#actions 保存) + BaseCard + a-form 分区 | ExpenseReimburseSubmit |
| 仪表盘/首页 | ~22 | KPI 条(StatCard) + 标签/卡片网格 + 图表 | express/Dashboard |
| 报表页 | ~10 | PageHeader(#toolbar 周期/组织) + ReportTable(金额/合计/数字右对齐) + ECharts 色板常量 | 待定（BalanceSheet 类，开工前定） |

> 前四类骨架与样板上游 PATTERNS.md 已定；本轮补齐每类模板「要消费的共享组件」（列表类的 DataTable/StatFilterTabs 是最大缺口）并把页面迁上去。报表页是否独立成第 5 类、配置页（system/\* / \*RuleManage）归列表还是表单，开工前各类定模板时确认。

### 4.2 每类的迁移循环（不大爆炸、不过度设计）

1. **定模板**：拿该类 1 个样板页跑通模板 = 该类骨架 + 它要消费的共享组件做实（与阶段0「建组件」是同一件事，按类组织）。
2. **批量套**：把该类其余页按同一配方迁移（含筛选条迁 `#toolbar`、StatusTag 替换、空态/密度收敛、裸 hex 清理——横切项随迁随做）。
3. **不一次性把 5 类模板都设计完**：每类先样板验证再批量，避免抽象出没人用的模板。
4. **套用顺序灵活**：模板按类定，但「先迁哪些页」可按业务优先级或模块排（如先 supplier/insurance/vehicle 练手 → 再 express/cardflow/finance 主力）。

### 4.3 横切项（迁移每页时顺手做）

- **裸 hex 工单按文件违规数降序**：`FlowDefinitionEditPage`(197) → `AmoebaPL`(73) → `AmoebaPLTemplate`(66) → `VoucherEntry`(60) → …（baseline 全量 1943）。
- **hex→令牌映射表**（统一口径）：文字 `#595959`/`#8c8c8c`→`--text-2`/`-3`；边框 `#e8e8e8`/`#e6e6e6`→`--border`；浅底 `#f5f5f5`/`#f8fafc`→`--bg-muted`/`--bg-page`；橙警示 `#fa541c`→`--color-primary`/`--color-warning`；蓝系（FlowDefinition `#1d4ed8` 等）按语义→`--color-info` 或彻底中性化为灰。无对应令牌再评估新增，**不就地造新 hex**。
- **递减度量**（PATTERNS 验收命令）：`<a-empty>` 91→↓、手写空态 43→↓、StatusTag 采用↑、冗余 `size="small"`↓、`.page-container` 覆写 45→0。
- 每类至少抽一页 preview 截图对范式（四类基线见 PATTERNS）。

## 8. 风险与缓解

- **大范围 hex 替换误判语义色 / 误清 ECharts**：按文件出工单 + 人工判定语义 + ECharts 显式白名单；先映射表后批量。
- **DataTable 迁移回归**：样板先行验证，逐页渐进迁移，不一次性全替；保留 a-table 直用的逃生路径。
- **PageLayout variant 改滚动链**：`table` 保留现有 flex 滚动链不变，`flow` 单独验证多卡片页（OrchestrationDetailPage 等）。
- **结构性重做（ContractList/CustomerDetail）动路由**：与纯收敛分离批次、单独 review；先出独立页/抽屉组件，再切换入口。
