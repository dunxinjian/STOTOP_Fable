# 登录页 / 主页 / Workhub —— UX 重设计（白纸版方向稿）

> 状态：**探索性愿景稿**，非已承诺的实现 spec。出发点是用户的提问"如果不考虑现有实现，你会怎么设计"。
> 本稿记录方向与取舍，落地需另起实现计划（plans/）。每一块都给**愿景版**与**务实版**两条线。
> 日期：2026-06-19

---

## 0. 背景与现状判断

STOTOP 是申通快递的内部经营/财务/审批中台，17 个一级模块，多组织（多网点账套）隔离，JWT + 组织上下文请求头。

现状关键事实（探查所得，作为重设计的基线）：

- **登录**：左右分栏（品牌 45% + 表单 55%）；JWT + RefreshToken；登录后弹**模态**选组织；进首页前有 **4 步过渡屏、最小 2300ms**；钉钉扫码可选；常驻一个绿色"系统正常"徽章。
- **主壳**：48px 顶栏（组织切换 / 工作台 badge / 前进后退 / 搜索 / 页签 / 刷新 / 全屏 / 用户）；**SmartSidebar 用"固定 + 最近"而非菜单树**；内容区 keep-alive 限 20。
- **Workhub**：三栏（发起 | 待办列表 tabs+筛选 | 详情 + 时间轴）；SignalR 实时；**操作可逆性分级**（可逆 5 秒撤销 / 不可逆二次确认）；J/K 键盘导航；8+ 数据源聚合（cardflow/oa/quality/task/contract/points/finance）。

**核心判断**：现状设计其实**已偏向"重度用户"**（工作台即主页、无菜单树、键盘流、可逆性分级——这些对每天泡在系统里的财务/审批是好东西）。但它有三个盲区，恰好对应"分层兼顾"的诉求：

1. 偶尔来的网点一线**会迷路**——"固定+最近"假设你已知道东西在哪，新人面对空待办列表不知"能干嘛"。
2. 管理者**没有驾驶舱**——老板登进来看到一堆待办，而非"今天这盘生意怎么样"。
3. 17 个模块**没有全景入口**——能搜、能从最近进，但无法"逛"。

---

## 1. 统一主张：「开机即决策，而非开机即菜单」

三个界面是**一条线**，不是三个孤立页面：

> **登录**（获取你今天的工作场）→ **主页**（角色自适应的"今天"）→ **Workhub**（把今天清零）

系统从登录第一秒起就回答："现在，对你这个角色，最该知道 / 最该做的是什么"，而把 17 个模块的全宇宙收进一条**安静的模块轨**——要逛随时在，但不挡路。

---

## 2. 脊柱决策（已与用户对齐：方案 B）

主壳骨架两种世界观，二选一决定后续所有界面：

- **方案 A · 纯工作台**：沿用现"工作台即一切"哲学，**无持久模块导航**，靠 ⌘K 命令面板 + 固定/最近。对老手极致干净，但对低频用户/管理者**没有空间感**、可发现性差。
- **方案 B · 模块轨 + 今天首页（已选）**：一条 ~46px 的**左侧模块轨**（17 模块图标，悬停展开）给空间永久性与可发现性 + 角色自适应"今天"首页 + ⌘K。**B 是 A 的超集**：老手照样全程 ⌘K 无视轨，三类人都不迷路，代价极小。

**结论：采用方案 B。** 顶栏精简、⌘K 上位为主导航、模块轨管"逛与发现"、横排"最近回到"。

---

## 3. 登录页

立场：**登录不是"进门"，是"获取你今天的工作场"**。打掉"输密码 → 死等 2.3s 剧场 loader → 弹窗选组织"的三段割裂。

**两条已与用户确认的决策：**
- ✅ **扫码 / SSO 扶正为主**（钉钉扫一扫第一位，账号密码退居其次）。前提：一线人人有钉钉且已绑定。
- ✅ **单组织用户跳过选择板，直接进主页**；仅多组织用户（财务/管理者）看到"工作场选择板"。

### 愿景版
两步、安静、单焦点：
1. **识别身份**：大幅二维码"钉钉扫一扫·一步登录"，"用账号密码"作次级折叠。品牌克制、留白多。
2. **今天的工作场（落地板）**：扫码后落到一块组织选择板，**每张组织卡片带实时状态**（待办 N · 异常 N · 上次进入时间），上次进入的组织预高亮、一个回车继续。
   - **关键转念**：原来那 2.3s 死等的 loader，变成一屏**有用的"选场子"**——加载时间不再浪费。

### 务实版（可直接落 Ant Design Vue）
保留左右分栏，修三个真痛点：
- **去掉 2.3s 剧场 loader**（要么真实进度、要么近瞬时）。
- **组织选择内联**进同一张卡（一行组织 chip，带待办数），不再弹窗。
- **系统状态只在异常时出现**（DB 不可用才提示并禁用并解释）；去掉常驻绿色噪音。
- 扫码为主 tab、账号密码次之；"记住我"默认勾选。

## 4. 主页（最重的一屏：角色分层自适应"今天"）

主页 = 一块**会读角色的"今天"画布**。登录后系统按角色/权限自动给你**对的那张脸**；身兼数职者用右上角「视角」一键切换；默认取最高频角色。

### 主壳 chrome（两版共有）
- **顶栏（精简）**：左 = 品牌点 + 组织切换；中 = **⌘K 命令栏**（搜索·跳转任意页·执行命令，主导航）；右 = 通知铃 + 我（头像菜单收纳刷新/全屏等）。
- **模块轨（~46px）**：home（当前）/ 待办 / 凭证 / 快递 / 经营 / 质控 / ⋯ / 设置；悬停展开模块名 + 子页。
- **内容区**：角色自适应"今天"。

### 三张脸
| 层 | 谁看到 | 第一屏给什么 |
|---|---|---|
| **决策层** | 老板 / 管理者 | **经营脉搏**：今日收入/毛利率/件量/待我决策 4 指标 + **异常雷达**（哪个网点/科目今天不对劲，一键钻取）+ **待我决策**（仅需老板拍板的高价值审批，非全量待办）。 |
| **执行层** | 财务 / 审批重度用户 | **待办流**："今天要清 23 条·预计 35 分钟" + 进入处理；待办流预览（决策前置卡）；我发起的在途单；快捷动作（凭证录入/跑计费）。 |
| **轻量层** | 网点 / 一线偶尔来 | **我要做什么**：4–6 个大按钮（上传数据/发起报销/查我的单/我的网点）+ "我交的单到哪了"状态追踪 + 罕见的"有 1 件事等你"醒目条。 |

角色→层映射由现有权限/菜单角色驱动。

### 愿景版 vs 务实版
| | 愿景版 | 务实版（直接落 Ant Design Vue） |
|---|---|---|
| 画布 | 流式自适应、留白多、一个"今天最该做的事"焦点 | 同样三层，做成**可增删拖拽的卡片网格**（每人自定义留哪些卡） |
| AI 一句话 | 有（经营摘要，如"毛利率较昨日 +2.1pt，2 网点签收率掉红线"） | 二期再加，先上**规则版**"今日提示" |
| 顶栏 | 几近清空，⌘K 独大 | 保留但瘦身（刷新/全屏收进 ⋯），⌘K 作为命令面板加上 |
| 执行层入口 | 待办流直接长在主页 | 主页给入口，点"进入处理"跳现有 Workhub（复用） |
| 落地 | 单组织用户登录后直接到这 | 同左 |

> **⚠️ 开放命门（待用户确认）**：决策层"经营脉搏"的数（今日收入/毛利率/件量）**能否准实时？** STOTOP 计费疑为"导入→跑批"，毛利可能须等结算才算得出。若如此，指标应诚实标成 **"最近已结算 6/18 / 昨日"**，宁可慢半天也别给老板一个假实时数。需逐口径确认：哪些当天可出、哪些必须 T+1。此问决定决策层这张脸是否成立。

## 5. Workhub 交互

立场：**Workhub 不是"待办列表"给你浏览，而是一条"把今天清零"的处理流。** 保留现状两个好东西并推到极致：可逆性分级 + 5 秒撤销、J/K 键盘流。

明确痛点：oa/cardflow 两套**泄漏的标签**、8 源**拖慢首屏**、卡片不够"决策即走"、批量太弱。

### 愿景版（借鉴 Superhuman"专注清收件箱"体感）
三个核心动作 + 专注模式：
1. **按业务实质聚类，不按来源**：用户永远看不到"OA审批/CardFlow审批"系统黑话，只看**业务类型**（费用报销/凭证复核/质量异常）；同类自动拢一起。
2. **智能批处理**："这 5 条标准报销金额均在权限内、单据齐全，可一键全过"——一个动作清掉一组。
3. **决策前置卡**：每张卡正面端出**那一个决策** + 你需要的 2–3 个事实 + **"为什么找我 / 我批了会怎样"**，让你基本不用打开详情面板。
4. **键盘全覆盖 + 专注模式 + 清零进度**：E 同意 / R 驳回 / T 转交 / J·K 切换 / U 撤销；专注模式一次一张全屏，像清一副牌；顶部"已清 18 · 还剩 5"进度条；清零后安静的"今天收工"空态。

### 务实版（对现有三栏"修缮"而非"重建"）
保留发起 / 列表 / 详情三栏骨架，修五个痛点：
1. **合并 oa/cardflow → 按业务类型**（去掉泄漏标签，解决残留混淆）。
2. **8 源懒加载 · 流式首屏**（每源骨架屏，cardflow/finance 先出，提速首屏）。
3. **卡片决策前置**（强化 WorkItemCard，详情面板变可选）。
4. **真批量**（超出 approve/reject：批量同类报销、批量复核凭证）。
5. **清零进度 + 补完 Enter=主操作 TODO**。
（保留 J/K、可逆性分级、SignalR。）

## 6. ROI 阶梯（落地优先级）

越上越便宜、收益越确定：

| 层级 | 改动 | 成本 | 收益 | 归属 |
|---|---|---|---|---|
| 🟢 立刻就赚 | oa/cardflow 合并去标签、顶栏瘦身、⌘K 命令面板、杀 2.3s loader、组织选择内联 | 纯前端+小后端 | 大 | 务实版 |
| 🟢 立刻就赚 | 模块轨 + 角色三层"今天"骨架（规则版，无 AI） | 中前端 | 大 | 两版共有 |
| 🟡 值得投 | 8 源懒加载流式、决策前置卡（各源补"为什么找我/批了会怎样"元数据）、真批量 | 中后端（每源改一点） | 大 | 务实→愿景过渡 |
| 🔴 重投入 | 业务实质聚类 + 智能批处理（聚类/规则引擎）、经营脉搏准实时、AI 一句话 | 大后端（取决于"今日毛利能否当天出"口径） | 高但不确定 | 愿景版 |

绿色两行**不管认不认同愿景都建议做**：近零风险，直接解决现状明确痛点（泄漏标签、首屏慢、登录割裂）。红色那行的风险点正是 §4 的数据口径命门。

## 7. 非目标 / 范围外（YAGNI）

- 不改技术栈（仍 Vue 3 + Pinia + Ant Design Vue + 现有 JWT/组织上下文/令牌系统）。
- 不动后端业务逻辑（计费三层、凭证规则、阿米巴口径等）——本稿只谈 UX 外壳与信息架构。
- 不做移动端重设计（现有 `/m/*` 独立布局另议）。
- 配色沿用现令牌系统（改色须改后端 DefaultConfigJson，本稿不涉及具体色值）。

## 8. 开放问题清单

1. **经营脉搏数据口径**（§4 命门）：哪些指标当天可出、哪些 T+1？决定决策层是否成立。
2. 扫码 SSO 覆盖率：是否真的人人有钉钉且绑定？影响登录主次。
3. 角色→层映射：现有角色/权限模型能否干净映射到 决策/执行/轻量 三层？多角色默认取哪层？
4. "决策前置卡"所需元数据（为什么找我/批了会怎样）：各数据源能否低成本提供？
