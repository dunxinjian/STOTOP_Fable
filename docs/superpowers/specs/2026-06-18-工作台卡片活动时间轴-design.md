# 工作台右栏：CardFlow 卡片活动时间轴（审批进度 × 操作日志合并）— 设计

- **日期**：2026-06-18
- **范围**：工作台右栏 `WorkHubDetail` 中 `source==='cardflow'` 工作项的详情，把「完整页面」的审批进度与操作日志合并进右栏，并按统一时间轴去重串联，减少跳页。
- **不在范围**：OA 审批（按最新设计 OA 模块已废除、由 CardFlow 全面取代，详见 [[oa-abolished-by-cardflow]]）；其它 source（quality/task/contract/...）详情维持现状；完整页 `CardDetailPage` 本身不改。

## 背景

现状：点击 CardFlow 工作项 → 右栏 `WorkHubDetail` 仅展示 WorkItem 自带字段（标题/摘要/元数据/操作），看审批进度与操作日志需点「查看完整页面」跳到 `CardDetailPage`（`/cardflow/cards/:id`）。体验割裂。

完整页数据现状（可直接复用）：
- 审批进度：`CardDetailPage.vue:561-562` 用 `CardTimeline` 渲染 `card.stageInstances` + `card.auditTrail`。
- 操作日志：`CardDetailPage.vue:578-586` 直接遍历 `logs`（`getCardLogs`）。

## 数据来源（均按 cardId 取，含时间戳可排序）

- `getCard(id)` → `CardDetailDto`（`web/src/api/cardflow.ts:219-220`）。含：
  - `stageInstances: StageInstanceDto[]`（`types/cardflow.ts:371-393`）：`stageName/type/status/round/activatedTime/completedTime/opinion/assignees[]`；`AssigneeDto`：`userName/status/opinion/completedTime`。
  - `auditTrail: CardFlowRuntimeAuditDto[]`（`types/cardflow.ts:355-369`）：`snapshotType('routeDecision'|'dynamicApprover')/stageName/title/reason/eventTime`。
- `getCardLogs(id)` → `ActionLogDto[]`（`web/src/api/cardflow.ts:264-266`）：`actionType/operatorName/operationTime/opinion`。
  - actionType 取值（`CardDetailPage.vue:374-391` logActionText）：create/submit/approve/reject/withdraw/resubmit/void/resume/countersign/transfer/cc/urge/autoComplete。
- WorkItem `metadata.cardId`（后端 `WorkHubService.cs:523-531` 注入），右栏据此取数。

## 去重合并模型（核心）

构建统一事件列表 `ActivityEvent[]`，**按时间升序**（发起在顶、当前待审在底）：

```ts
interface ActivityEvent {
  time: string                         // ISO 时间戳
  kind: 'op' | 'decision' | 'node-enter' | 'system'
  title: string
  actor?: string                       // 操作人/审批人
  node?: string                        // 所属节点名
  opinion?: string                     // 意见
  status?: string                      // decision: approved/rejected/...
}
```

构建规则（三源互不重复）：
1. **操作事件（op）** ← `actionLog`，**过滤掉 `approve`/`reject`**（决策改由审批节点出，去重）。其余（create/submit/withdraw/resubmit/void/resume/countersign/transfer/cc/urge/autoComplete）→ `{time: operationTime, kind:'op', actor: operatorName, title: logActionText(actionType), opinion}`。
2. **节点进入（node-enter）** ← `stageInstances` 中有 `activatedTime` 者 → `{time: activatedTime, kind:'node-enter', title: '进入「'+stageName+'」', node: stageName}`。
3. **审批决策（decision）** ← `stageInstances[].assignees` 中有 `completedTime` 者 → `{time: completedTime, kind:'decision', actor: userName, node: stageName, status, opinion, title: statusLabel(status)}`。
   - 边界：节点无 assignees 但有 `completedTime`（自动节点）→ `{time: completedTime, kind:'decision', node, status, title:'自动完成'}`，不丢自动完成。
4. **系统轨迹（system）** ← `auditTrail` → `{time: eventTime, kind:'system', title: title || reason}`。
5. 合并后按 `time` 升序稳定排序；丢弃无有效 time 的事件。
6. **当前待审 footer**（不入排序列表，渲染在最底高亮）：`stageInstances` 中 `status ∈ {pending, processing}` → 「等待「stageName」审批 · 审批人：{pending assignees 的 userName 拼接}」。

## 组件设计

### 新增 `web/src/components/workhub/CardActivityTimeline.vue`
- Props：`cardId: number`。
- 行为：`watch(cardId, immediate)` → 并行 `getCard(cardId)` 与 `getCardLogs(cardId)`（各自 `.catch` 兜底为安全默认，互不拖累）→ 构建 `ActivityEvent[]` + 待审 footer。
- 状态：`loading`（骨架/`a-spin`）、空态（无任何事件时「暂无活动记录」）、错误（任一接口失败仍尽量渲染已得到的部分）。
- 渲染：竖向时间轴（左侧轴线 + 节点圆点），每条按 kind 给图标/色；decision 含节点名 + 意见块；system 标「系统」；footer 高亮当前待审。
- 轻量缓存：模块级 `Map<number, {detail, logs}>`，cardId 命中则直接用，避免 J/K 来回切重复请求（预览态可接受短暂陈旧；用户在右栏执行通过/驳回后该工作项会离开列表，不存在长期陈旧）。

### `WorkHubDetail.vue` 集成
- 新增 computed `cardId`：`source==='cardflow'` 时取 `Number(workItem.metadata?.cardId)`，否则 null。
- 详情体：当 `cardId` 有值 → 在标题/摘要之下、操作按钮之上插入 `<CardActivityTimeline :card-id="cardId" />`，**替代** cardflow 工作项原本的通用「元数据 grid」（其它 source 仍走原通用渲染）。
- 「查看完整页面」链接保留为兜底（完整表单/附件仍在完整页）。底部 sticky 操作按钮不变。

## 视觉与复用
- 复用 `CardTimeline.vue` 的语义：状态色（approved→`var(--color-success)`、rejected→`var(--color-danger)`、processing→`var(--color-info)`、pending→中性灰）、状态文案（已通过/已退回/处理中/待处理/已跳过）、节点类型图标（approval→TeamOutlined、auto→RobotOutlined、cc→SendOutlined）。
- **新组件 JS 色值优先用 `var(--*)` 令牌**（不照搬 CardTimeline 里的裸 hex `#fa541c/#8c8c8c`）；`<style>` 一律令牌、无裸 hex（满足 stylelint）。
- 时间格式复用 `dayjs`（与现有组件一致；列表可显示 `MM-DD HH:mm`）。

## 测试与验证
- 无前端测试运行器（见 [[frontend-typecheck-baseline-red]]）：`cd web && npm run build` 编译门禁 + `npm run lint:style` 新样式无裸 hex。
- 手动（preview，用户在已登录浏览器）：
  1. 点 CardFlow 卡 → 右栏出现合并时间轴：发起/提交（op）→ 进入节点 → 审批决策（含意见）→ 系统条件流转 → 底部当前待审；**通过/驳回不重复出现**。
  2. J/K 切换不同卡 → 时间轴随之更新；来回切命中缓存不重复请求（网络面板核对）。
  3. 无活动的卡 → 空态；某接口失败 → 仍渲染另一部分、不白屏。
  4. 「查看完整页面」仍可跳完整页；底部 通过/驳回 操作正常。

## 风险
- `actionType` 去重集合定为 `{approve, reject}`；若后端新增其它"决策类"动作类型，需同步加入排除集（在组件内集中常量 `DECISION_ACTION_TYPES`）。
- 自动节点（无 assignees）的完成事件依赖 stage `completedTime`，已在规则 3 边界覆盖。
- 右栏按 cardId 取数为新增网络调用；以缓存 + loading 降低 J/K 抖动；不引入对 useWorkHub 的耦合。
