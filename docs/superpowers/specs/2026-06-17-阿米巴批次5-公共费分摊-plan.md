### S2 — 月度分摊接入 ✅ 已落(本会话)
- `IsCommonCostLeaf`(F分摊方式=="volume")；`ResolveVolumeIndicatorIds`(发件/派件票量 indicator FID)。
- 子报表(IsSubReport)：内部额外跑一次无 scope 全口径聚合 → `fullTotals`(公共费叶全额)+`fullScopeVolume`；请求级缓存。

**实施记录(对设计的修订/落法)**：
- `AmoebaPLService` 新增 4 个 helper：`IsCommonCostLeaf`/`ResolveVolumeIndicatorIds`/`ComputeVolumeBasis`(均 internal static 纯函数) + `ApplyCommonCostAllocation`(internal 实例，不碰 DB)；构造注入 `ICommonCostAllocationEngine`。
- **纠正设计 §4.5 悲观假设**：多期路径聚合本身无 SQL 级 scope（`AggregateBillingRevenue(...,null,null)`），scope 是 `ApplyScopeFilter` 内存后置过滤且返回新列表。故全口径基线 = 过滤前的 `all`，**零额外 DB 聚合**即得 `fullTotals`/`fullScopeVolume`，无需"+1 次全表聚合 + 请求级缓存"。
- **件量口径**(`ComputeVolumeBasis`)：send=billing `SUM(WaybillCount)`(Source=="billing" 天然出港)→回退发件票量 indicator 匹配值；deliver=派件票量 indicator 匹配值(进港无 billing 源)。scope/全口径各算一次(同 fullPoints/scopedPoints)。
- **子报表跳过公共费叶**改为**注入后处理**(非 Match 短路)：公共费叶值=分摊额(命中)或移除(全额缺失)，避免 send 类公共费随 scope 存活留下直接归段残值；好处是不污染 unmatched 告警。注入点在「指标并入之后」(ComputeVolumeBasis 需读 matched 内票量 indicator 值)。
- 注入仅子报表触发；全口径(scope=null)`periodFullScopePoints[i]=null`，公共费叶走正常匹配不注入。
- 单测：`AmoebaCommonCostWiringTests`(helper 纯函数 10) + `AmoebaPLServiceTests` 2 例(比例分摊 600=1000×300/500、全额缺失移除残值)。Finance 套件 58 全绿。
- **未做**(留后续切片)：MatchDataPointsToPLItems 的 `skipLeafIds` 参数(设计 §4.4 的 Match 短路)未引入——注入后移除已等价覆盖；Reconcile 闭合自检尚未在活跃路径调用(引擎 `Reconcile` 已就位，待多 scope 全集汇总场景接入)。

### S3 — granularity（批次4余项）✅ 已落(本会话, 3 子提交 + 审查修复)
- `PeriodToDateRange(period, granularity)` 按 §5.3 表（day/week/month/quarter/year，半开区间）；`CalcPrevious/CalcYoy(period, granularity)` 泛化。

- `BuildPeriodDataPoints(period, granularity)` 选源：month=voucher+dep+billing；day/week=estimate。**C1 互斥**：month 不 AddRange(estimate)。

**实施记录(对设计的修订/落法)**：
- **S3a 周期方法泛化(28c8dfd)**：4 方法加 granularity 参(默认 month 兼容) + 新增 `BuildPeriodKey/NormalizeGranularity/IsLedgerGranularity` + ISOWeek 解析。**区间约定**：返回各粒度「闭区间末日」，聚合内部 `< end.AddDays(1)` 自动转半开，三聚合方法零改动。day/week 同比返 null。
- **S3b FPeriodKey(14cf2ff)**：实体加可空 `FPeriodKey`(列 F期间键) + 迁移 **V7**(幂等加列+回填 `M:`+期间，分两 batch)；**唯一索引保持 F期间不变**(各粒度期间串本就唯一；改索引会踩 schema 管线"回填前用全 NULL 键建唯一索引撞 NULL"陷阱)。CRUD(SaveEstimate/SaveManual)写 FPeriodKey。
- **S3c 接入(0da1da7)**：DTO 加 `Granularity`(默认 month，前端零改动兼容)；抽 `SelectSources`(C1 互斥纯函数) + `BuildPeriodDataPoints`；manual 加载/去重 + 估值取数全改按 periodKey，全部 manualLookup 消费方穿线。**源选择修订**：day/week = billing + estimate（非纯 estimate；billing 按日期可裁，解 S4 件量依赖），ledger = billing+voucher+dep。
- **对抗式审查修复(ec9279d)**：[HIGH] billing SQL `<=末日` → 半开 `<末日+1`(与 voucher 一致；day 粒度否则整日漏带时分单)；[LOW] UPSERT update 分支补写 FPeriodKey 自愈；[LOW] 死代码 SumLeafDescendants 参数改名一致化。
- **C1 当月例外(60695e0，用户 2026-06-17 修正)**：估值是当月未结期(凭证不全)的临时填充，月结后由真实凭证取代。`SelectSources(granularity, periodClosed)` 时间感知 + `IsPeriodClosed(末日<今天)`：台账粒度仅**未结**(当月/未来)叠加 estimate、**已结过往**月份用 voucher+depreciation；日/周恒用 estimate。**顺带解决审查 HIGH#2**：当月估值按 `M:`+期间 键被当月月报读到(天然归宿)，不再"消失"。
- 单测：`AmoebaPeriodGranularityTests`(周期纯函数+SelectSources 当月/已结×粒度+IsPeriodClosed，~40 用例) + `AmoebaPLServiceTests`(FPeriodKey 写入/自愈)；Finance 套件 126 全绿，WebAPI 构建通过。
