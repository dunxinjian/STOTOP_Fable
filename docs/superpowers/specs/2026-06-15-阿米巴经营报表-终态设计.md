## 0. 背景与目标

### 0.1 报表矩阵(同一损益项模板 + 不同 scope)

| 报表 | Granularity | Scope | 说明 |
|---|---|---|---|
| ① 太仓美申全口径 | month | `null` | 现有口径,无作用域 |
| ② 城区/南郊/沙溪/浏河 各一张 | month | `{Outlets:[单网点]}` | 网点子报表,含分摊后真实净利 |
| ③ 裹裹项目独立核算 | month | `{Projects:["裹裹"]}`(不带 Outlets) | 跨4网点汇总全公司裹裹业务 |
| ④ 太仓美申出港 | month | `{Directions:["OUT"]}` | 仅出港方向 |
| 任意上表的日/周钻取 | day/week | 同上 scope + 切粒度 | 估算源 |

矩阵正交性:**scope 与 granularity 互相独立**——③裹裹既可月度(凭证源)也可日/周(估算源)。这是把两者都并入 `AmoebaMultiPeriodRequest` 的价值。

### 0.2 方案B = 业务方向一等公民

出/进港及所有维度切片靠**辅助核算**(值挂在凭证分录上)过滤,不靠物理科目码。同一科目(如 540104 操作)同时服务出/进港,用 filters 的 `business_direction` 区分,**不造重复末级码**。

### 0.3 两层过滤模型

- **L1 报表作用域 scope**(运行期/请求级):对全表统一过滤。must-have = 网点 outlet / 项目 project / 业务方向 direction;可扩展 = 经营单元 unit / 品牌 brand。
- **L2 损益项内在维度**(配置期/种子):损益项 V2 filters 写死内在维度(出港操作=540104∧OUT;揽件员工资=5601010101∧部门)。
- **叠加 = AND**:网点X的出港操作 = `dp.AccountCode StartsWith "540104"` ∧ `AuxValues["business_direction"]=="OUT"`(L2) ∧ `dp.SiteCode=="X"`(L1)。

### 0.4 无数据前提的工程含义

无正式凭证 → ① 物理删列/删迁移;② 种子直写正交列,不靠 Migrate 反推;③ 改匹配语义无回归对象;④ **端到端验证须先造带 business_direction 标的测试凭证**(批次2 闸门),否则只能靠单测。

### 2.2 scope 执行点 = 独占匹配之前预过滤(关键抉择)

**裁决:scope 在 `MatchDataPointsToPLItems`(`PL.cs:894`)之前对每期 DataPoint 池做预过滤,而非之后。** 理由:

1. **独占正确性**:voucher 分支 `pool.Remove`(`PL.cs:987`)。若后置过滤,靠前损益项会先把不属本 scope 的点 Remove,导致 `unmatchedWarnings`(`PL.cs:287-294`)口径错乱。
2. **件量分母正确**:billing 不移除(`PL.cs:983`),件量分摊分母须是本 scope 件量而非全公司(决策4)。
3. **改动面最小**:只在"聚合完 → match 之前"插纯函数,不动 match 内 700 行。
4. **引用语义自洽(冲突C2)**:scope 用 `.Where().ToList()` 产新 List 但**不 clone DataPoint**(仍同引用),Match 内独占用 `consumed` HashSet(引用相等),两者共存无冲突。

```csharp
private static List<DataPoint> ApplyScopeFilter(List<DataPoint> points, AmoebaReportScope? scope)
{
    if (scope == null) return points;
    var filters = new List<AuxFilter>();
    if (scope.Outlets?.Count   > 0) filters.Add(new(){AuxType="outlet",             Codes=scope.Outlets});
    if (scope.Projects?.Count  > 0) filters.Add(new(){AuxType="project",            Codes=scope.Projects});
    if (scope.Directions?.Count> 0) filters.Add(new(){AuxType="business_direction", Codes=scope.Directions});
    if (scope.Units?.Count     > 0) filters.Add(new(){AuxType="business_unit",      Codes=scope.Units.Select(x=>x.ToString()).ToList()});
    if (scope.Brands?.Count    > 0) filters.Add(new(){AuxType="express_brand",      Codes=scope.Brands});
    if (filters.Count == 0) return points;
    return points.Where(p => AuxiliaryMatches(p, filters)).ToList();  // 复用同一 AND 语义
}
```

**严格剔除口径**:当某维度被约束时,该维度为空的 DataPoint 被剔除(沿用 billing 单品牌严格语义)。这对子报表正确性必须——否则网点子报表掺入无网点归属的全公司公共费。**例外**:公共费分摊在 scope 预过滤之后单独执行(§4)。

### 4.3 分摊算法(核心)

公共费全额是全公司发生的,按 `scope件量 / 全口径件量` 分到子报表。**分子分母必须同 basis 配对**(折旧 total、出港操作费 send,禁 send 分子配 total 分母)。

```

```

接口与实现:

```csharp
public interface ICommonCostAllocationEngine
{
    Dictionary<long, decimal> Allocate(
        IReadOnlyList<FinAmoebaPLItem> commonCostLeaves,
        IReadOnlyDictionary<long, decimal> fullScopeTotals,   // 全口径每公共费叶 FID 发生额
        VolumeBasis scopeVolume, VolumeBasis fullScopeVolume,
        out List<string> warnings);

// 实现要点
foreach (var leaf in commonCostLeaves) {
    if (!fullScopeTotals.TryGetValue(leaf.FID, out var total) || total == 0m) continue;
    long volScope = scopeVolume.ByKind(leaf.F分摊基数);
    long volAll   = fullScopeVolume.ByKind(leaf.F分摊基数);
    if (volAll == 0) { warnings.Add($"公共费[{leaf.FItemName}]全口径{leaf.F分摊基数}件量为0,本期记0"); result[leaf.FID]=0m; continue; }  // 除零返0

}
```
