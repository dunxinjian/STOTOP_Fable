# 报价模块视觉债清理 — 设计（spec）

> Phase 2 痛点页第 1 项。承接「功能页逐模块巡检与收敛」，对**报价模块**做**视觉一致性清理**：把裸 hex 颜色字面量换成新令牌、对齐「中性墨 + 橙点缀」方向。
> 用户确认痛点**仅为 C（视觉债）**——**不动布局、交互、计费逻辑、数据**（A 内容藏抽屉 / B 信息条密杂 / D 矩阵交互均未选）。

## 1. 背景与目标

报价编辑页（QuotationEdit）被点名为最烂页之一。逐文件扫描（18 文件并行）实测：

- 「旧 SCSS 变量」**不是债**——`variables.scss` 已把 `$text-regular`→`--text-2`、`$color-primary-bg`→`--color-primary-light`、`$font-size-sm`→`--font-sm` 等桥接到新令牌。
- **真正的债 = 裸 hex 字面量，约 87 处 / 16 文件**，含三类位置：`<style>` 块、模板内联 `style=""`、以及 `render`/`h()` 函数里的 style 字符串（后两类 **stylelint 抓不到**，是这页"看着旧"的主因）。
- 其中夹带**旧主色蓝残留**（`#f0f7ff`/`#eef3fa`/`#f0f4f8`/`#f7f9fc`/`#bae0ff`/`#f0f5ff`/`#f0f9ff`）与 **4 条蓝渐变**（WeightSegmentEditor 标尺段）。

目标：报价模块全部裸 hex → 新令牌，旧蓝清零、渐变扁平，视觉与全站「中性墨 + 橙点缀」统一；行为/布局/逻辑零改动。

## 4. 设计决策（D1–D5，已确认）

- **D1 矩阵斑马**：**保留斑马但中性化**——偶行 `var(--bg-card)`（白）/ 奇行 `var(--bg-page)`（`#F7F8FA` 极淡，即旧蓝 `#f7f9fc` 的落点）；行 hover 用更深一档 `var(--bg-muted)`（与奇行可区分）。区域合并格（左侧跨行列，旧 `#fafafa`/`#f0f4f8`）**恒 `var(--bg-muted)`，不随奇偶变**。属列表标准里「超宽超密表按页例外恢复斑马」的允许例外。
- **D2 旧蓝高亮 → 中性 / 橙**：
  - 行 hover `#f0f7ff`、段卡 hover `#f0f9ff` → `var(--bg-muted)`（中性，对齐全局 hover）。
  - 价格列头 `#eef3fa` → `var(--color-primary-light)`（价格=核心数据，浅橙点缀；普通列头仍 `--bg-muted`）。
  - 已覆盖单元格 `#f0f5ff`（PriceCellInput `.has-override`）→ `var(--color-primary-light)` + 橙点（持久状态用橙）；**同时把该单元格 hover 改 `var(--bg-muted)`**，使「hover（中性）/ 已覆盖（橙）」可区分。
- **D3 重量段标尺渐变（WeightSegmentEditor 4 条）**：**扁平化去渐变**——普通段 `var(--bg-muted)`、hover/激活段 `var(--color-primary-light)`、无穷段渐隐改纯 `var(--bg-muted)`。改完 **live 复核微调**。
- **D4 色块上的白字**：橙/红色块上的图标白字 `#fff`（FixedPriceCostMatrix 拆分/合并按钮 hover、WeightSegmentEditor 气泡/×按钮、WeightSegmentPanel ×按钮等）→ **新增令牌 `--text-on-accent: #FFFFFF`** 替换。
- **D5 错误焦点环**：PriceCellInput `.has-error:focus` 的 `rgba(255,77,79,.15)`（旧红）→ **新增令牌 `--color-danger-border`**（仿现有 `--color-primary-border`），统一焦点环。

## 5. 新增令牌（2 个）

在 `theme.ts applyDesignTokensCSS()` 注入 + `variables.scss` 桥接（与 `--border-faint` 同套路）：

- `--text-on-accent: #FFFFFF` — 强调色块（橙/红/危险）之上的文字/图标色。SCSS 桥 `$text-on-accent`。
- `--color-danger-border: rgba(214, 88, 78, 0.30)` — 危险态焦点环/描边（`--color-danger` #D6584E 的 30% 透明，对齐 `--color-primary-border` 的 .30 口径）。SCSS 桥 `$color-danger-border`。

写完后同步进 `web/docs/TOKENS.md` 与 PATTERNS §三令牌速查。

## 6. 实施约定

- **统一用 `var(--token)`** 落地——`<style>` 块、模板内联 `style=""`、`render`/`h()` 字符串三处都用 `var()`（`var()` 三处都在运行时生效，且对齐设计系统「组件消费 CSS 变量」方向）；不引入 SCSS `$` 新写法、不混用。
- **渐变扁平**：`linear-gradient(...)` 整条替换为单一 `background: var(--token)`。
- **render/inline 字符串逐处替换**：ShopPanel / BatchShopModal 的 `h()` 表格样式、各处内联 `style` 里的 hex 逐个换 `var()`；**不**为省事抽成 class（保持 diff 小、YAGNI；结构不动）。
- **不动**：模板结构、组件拆分、事件/交互、计费与数据、布局间距。
- 暗色模式不专门处理（沿用现状 dark-ready）。

## 7. 验证

- `npm run build` 绿（vite，非 type-check）。
- `npx stylelint` 改动文件 0 报错。
- **关键**：stylelint 只检 `<style>` 块，**抓不到内联 / render 字符串里的 hex**——故收尾用 `rg "#[0-9a-fA-F]{3,6}"` 复扫报价模块这 16 文件，确认裸 hex（含旧蓝）**清零**（保留的仅 `--text-on-accent` 令牌定义处的 `#FFFFFF`、及 echarts/合法豁免若有）。
- 用户 live（9001 HMR）看三处：**矩阵**（斑马/价格列头/hover/已覆盖）、**重量段标尺**（渐变扁平）、**冲突弹窗表格**（ShopPanel/BatchShopModal render 表）。

## 8. 非目标 / 风险

- 非目标：A（内容藏抽屉）、B（信息条密杂）、D（矩阵交互）——本轮不碰；如后续要做另起。
- 风险：render 字符串里 `var()` 在极少数无 CSS 上下文场景（如导出/邮件 HTML）不解析——本模块这些表格都在应用内渲染，无此场景，安全。
- 风险：D3 渐变扁平后标尺观感变化较大，以 live 复核为准，必要时回调浓淡。
