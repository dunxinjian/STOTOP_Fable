# 毛利分析页（ProfitAnalysis）重做 — 设计（spec）

> Phase 2 痛点页第 2 项。用户确认痛点为 **A（视觉债）+ B（KPI 条改扁平）+ C（7 表合一）**。
> 铁律:**不动取数 API、不动计费/口径、不动各视角的列与业务含义**;只做「视觉对齐 + KPI 扁平 + 模板去重」,**所有行为 1:1 不变**。

## 1. 背景与目标

`web/src/views/express/report/ProfitAnalysis.vue`(单文件 ~1062 行)是 7 视角毛利报表(业务对象/报价/中间人/业务员/重量段/流向 + 趋势图)。问题:① 视觉债(KPI 蓝灰渐变、本页 scoped 表格覆盖把全局中性样式盖回旧样、图表色板老旧、散落裸 hex/旧红);② KPI 条用渐变底不合「中性墨 + 橙点缀 + flat」;③ 6 个表格视角是 6 段近乎重复的 `<a-table>`,模板冗长难维护。

目标:视觉对齐全站 + KPI 扁平化 + 6 表合一为配置驱动的单表,行为完全不变。

## 2. 范围

单文件 `ProfitAnalysis.vue`(无子组件,自包含)。不新建组件。

## 3. A — 视觉债清理

- **删本页 `:deep(.ant-table)` 覆盖里重置旧样式的条目**(`<style>` 内 .ant-table-thead th `background:#fafafa`、tbody hover `background:#f0f7ff`、summary `background:#fafafa`)——让全局 `ant-override.scss` 的中性表头/中性 hover/耳语行线接管。仅保留本页确需的非颜色微调(若 font-size 13/padding 与全局冲突则一并交回全局,默认全删颜色覆盖)。
- **裸 hex / 旧红 → 令牌**(`<style>` 块):
  - `.kpi-label` `#8c8c8c` → `var(--text-3)`
  - `.kpi-divider` `#e0e0e0` → `var(--border)`
  - `.val-caution` `#d4a106`(低正毛利 10-20%，比 `.val-warning` 深一档）→ `var(--color-warning-text)`（深琥珀，保留与 val-warning 两档区分；勿并入 `--color-warning` 否则两档撞色）
  - `.kpi-clickable:hover` `rgba(255,77,79,0.06)` / `.kpi-active` `rgba(255,77,79,0.1)` → `var(--color-danger-light)`(浅红底)
  - `.kpi-active` `box-shadow inset … #ffa39e` → `var(--color-danger-border)`
  - `.trend-view` `background:#fff` → `var(--bg-card)`
- **图表色板换新**(echarts `renderTrendChart` 的 `setOption`;canvas 不解析 `var()`,用真 hex 是既定豁免,但值要对齐新板):
  - 应收线 `#3A6FB0` → `#5B7290`(墨蓝灰 info)
  - 成本线 `#E5484D` → `#D6584E`(danger)
  - 毛利线 `#2BA471` → `#3E9E6E`(success);面积渐变 `rgba(82,196,26,*)` → 毛利绿低透明 `rgba(62,158,110,0.22)`→`rgba(62,158,110,0.02)`
  - 毛利率柱 `#E6A700` → `#D49A2E`(warning)
  - 轴标签 `#666`/`#999` → `#5A6068`/`#8A9099`(=text-2/text-3 的值)

## 4. B — KPI 条扁平化（方案 1）

- `.kpi-strip` 去 `linear-gradient(135deg,#f6f8fc,#eef2f9)` → `background: var(--bg-card)` + `border: 1px solid var(--border)`(扁平中性底,直角/8px 圆角沿用)。
- 保留:一行密排布局、`.kpi-divider` 分隔、毛利红绿信号(`--color-success`/`--color-danger`)、可点的「亏损对象/零应收」告警块(底色用 `--color-danger-light`)。
- 不改 KPI 的计算逻辑与条目(`activeSummary`/`clientAlertStats` 等照旧)。

## 5. C — 6 表合一（配置驱动单表）

把 6 个表格视角(client/shop/intermediary/salesman/weightSegment/region)的 6 段 `<a-table>` 收敛为 **1 个动态 `<a-table>`**;**趋势图(trend)保持独立 `v-if` 图表**。

**5.1 列 format 元**:给各视角 columns 的数值/特殊列加 `format` 字段,驱动统一渲染:

| format | 渲染 |
| --- | --- |
| (无 format) | 该列不进 #bodyCell 分支，由 a-table 默认按 dataIndex 渲染(与现状一致：纯文本列如 clientName/region/weightSegment/networkPointId 本就不在 bodyCell) |
| `num` | `formatNum(v)` 整数千分位 |
| `weight` | `v.toFixed(2)` |
| `money` | `¥v.toFixed(2)` |
| `profit` | `¥v.toFixed(2)` + `val-positive/negative` 类 |
| `rate` | `formatRate(v)` + `profitRateClass(v)` 类 |
| `tagClient` | 业务对象视角 clientType 标签(蓝/绿/默认) |
| `tagIntermediary` | 中间人视角 clientType 标签(业务代理/承包区/其他) |

(业务员 `networkPointId` 为纯文本;`commissionIncome`/`avgCommission` 用 `money`。)

**5.2 viewConfig 注册表**:

```ts
// 静态:每视角的列/行键/分页/滚动/特性标志
const VIEW_CONFIGS: Record<string, {
  columns: any[]; rowKey: string | ((r:any)=>string);
  pagination: any; scroll: Record<string, any>;
  hasSummary?: boolean; hasExpand?: boolean;
}> = {
  client: { columns: clientColumns, rowKey: 'clientId', pagination: PAGE50, scroll: {x:1100,y:TBL_Y}, hasSummary: true },
  shop:   { columns: shopColumns, rowKey: shopRowKey, pagination: PAGE50, scroll: {x:1000,y:TBL_Y} },
  intermediary: { columns: intermediaryColumns, rowKey: 'clientId', pagination: PAGE50, scroll: {x:1100,y:TBL_Y} },
  salesman: { columns: salesmanColumns, rowKey: 'salesmanId', pagination: PAGE50, scroll: {x:800,y:TBL_Y} },
  weightSegment: { columns: weightSegmentColumns, rowKey: 'weightSegment', pagination: false, scroll: {x:900,y:TBL_Y} },
  region: { columns: regionColumns, rowKey: 'region', pagination: false, scroll: {x:1000,y:TBL_Y}, hasExpand: true },
}
// data/loading 按视角取
const dataMap = { client: displayedClientData, shop: shopData, ... }   // 注:client 用「过滤后」displayedClientData
const loadingMap = { client: clientLoading, ... }
const currentView = computed(() => ({ ...VIEW_CONFIGS[activeView.value], data: dataMap[activeView.value].value, loading: loadingMap[activeView.value].value }))
```

`PAGE50`/`TBL_Y`/列定义等沿用现值(列 format 元为新增,值不变)。

**5.3 模板**:

```vue
<a-table v-if="activeView !== 'trend'"
  :columns="currentView.columns" :data-source="currentView.data" :loading="currentView.loading"
  :pagination="currentView.pagination" :row-key="currentView.rowKey" :scroll="currentView.scroll" size="small"
  :expanded-row-keys="currentView.hasExpand ? expandedRegionKeys : undefined"
  @expand="currentView.hasExpand ? handleRegionExpand : undefined">
  <template #bodyCell="{ column, record }"> <!-- 按 column.format 统一渲染 --> </template>
  <template v-if="currentView.hasSummary" #summary> <!-- client 合计行(原样) --> </template>
  <template v-if="currentView.hasExpand" #expandedRowRender="{ record }"> <!-- region 省份明细表 --> </template>
</a-table>
<div v-else class="trend-view"> <!-- 趋势图,原样 --> </div>
```

- 省份明细内表(expandedRowRender)同样用 format 元 + 统一渲染逻辑(`provinceColumns` 加 format)。
- 用**普通 `<a-table>`(非 DataTable)**——报表需合计行/展开/无序号列、分页口径各异;靠全局中性表格样式呈现。

**5.4 行为 1:1**:排序(sorter/defaultSortOrder)、合计行(仅 client)、省份展开懒加载(仅 region)、亏损过滤(showLossOnly→displayedClientData)、导出、各列宽/对齐/fixed——全部保持不变。逐视角 live 验。

## 6. 实施约定

- `<style>` 块替换值用 `var(--token)`;echarts `setOption` 用新板真 hex(canvas 豁免)。
- 不动 `<script>` 里取数/计算/口径;C 仅新增 `format` 列元 + viewConfig/currentView + 改模板,不改各 `fetchX`/`xSummary`/`xColumns` 的数据语义。
- 保留趋势图独立渲染与 echarts 生命周期(init/dispose/resizeObserver)。

## 8. 非目标 / 风险

- 非目标:不重构取数 API、不碰计费口径、不改列业务含义、不引入 DataTable。
- 风险:C 合并易引入行为回归(排序/合计/展开/过滤)。缓解:viewConfig 保持各视角原参数原样、逐视角 live 验;计划上 **A+B 先落(低风险可视)、C 后落**,C 出问题可单独回退不影响 A+B。
