## 0. 现状结论（已亲自核验，非臆测）

### 0.2 已排除的误报（智能体报错，我核验后否定——避免修不存在的 bug）

- ❌ ~~"所有 Service 创建时 FOrgId 未初始化 = P0 组织隔离漏洞"~~ → **假**。`STOTOPDbContext.SaveChanges` 对 `EntityState.Added` 且 `FOrgId==0` 的 `IOrgScoped` 实体**自动注入当前组织**（STOTOPDbContext.cs:197-201）。组织隔离正常。
- ❌ ~~"9 个视图不可达、只有 1 条路由"~~ → **误导**。本系统路由是**菜单驱动动态生成**（`permission.ts:generateRoutes`→`buildRoutes`→`componentPathToModule` 用 `import.meta.glob` 把 `dormitory/BuildingManage` 解析为真实组件）。菜单种子 FID 701-708 已定义 8 个页面 → 只要种子在库里，8 页运行时自动挂路由。真实缺口是更窄的 R1/R2（RoomManage 子页 + 主从 name 导航 + 种子是否进库）。
- ❌ ~~"R4 宿舍视图无 orgSwitchVersion watch = 切组织不刷新"~~ → **假**。`MainLayout.vue:22` 给 `<router-view>` 组件加 key=`${orgSwitchVersion}:${pageRefreshVersion}:${fullPath}`，组织切换时 `orgSwitchVersion++` → key 变 → **整树重挂 → onMounted 重跑 → 自动重拉**。9 个宿舍视图均在 onMounted 拉数 → 已自动受益。加逐视图 watch 属冗余 cargo-cult，不做。（AmoebaPLTemplate 显式 watch 是其作为嵌套子组件的特例。）

## 1. 路由架构（已厘清）

- 静态路由：`routes.ts` 的 `staticRoutes`/`layoutRoute.children`，建 router 时注入。
- 动态路由：登录后 `loadDynamicRoutes()` 取后端菜单 → `generateRoutes()` → 对每个 `type:'menu'` 且有 `route`+`componentPath` 的项，`componentPathToModule()` 经 glob 解析组件 → `router.addRoute('Layout', route)`。动态路由 `name = 菜单 code`。
- 成熟模块（finance）**双保险**：既在菜单种子定义（驱动侧栏 + 动态路由），又在 `layoutRoute.children` 静态定义（提供稳定 name `AccountManage`、隐藏带参详情页 `finance/budget/editor/:versionId` `hidden:true`，并在种子缺失时兜底）。已确认 finance 同路径静态+动态双份共存、生产可用。

## 2. 目标与验收标准（"可正式交付"的定义）

1. **功能完善**：设计文档 17-dormitory.md 全部职责可用——层级资源（楼栋→房间→床位）、入住/退宿、费用录入+分摊、设施、报修工单全流程、访客、卫生检查、统计看板；所有 CRUD 与状态流转闭环（含编辑）。
2. **集成无缝**：侧栏菜单可进每一页；主从页互跳正常；切组织即时刷新；所有前端 API 调用都命中真实后端端点（零 404）。
3. **UX 非常好**：统一令牌配色（零裸 hex）；每页有加载/空/错三态；表单完整校验+友好反馈；危险操作二次确认；分页/筛选/响应式；与成熟模块观感一致。
4. **质量过硬**：后端有服务级+组织隔离+集成测试且全绿；前端 type-check 不引入新红、`vite build` 通过；真机逐页走查零控制台错误。
5. **可交付**：迁移/种子可重放；权限落地符合系统约定；代码符合本仓约定（F 前缀、令牌、命名）。

## 4. 测试与验证策略

- 后端：xUnit + 内存/独立 SQLite 或测试库；**测试间隔离**，绝不复用会被并发清空的共享库（记忆 org192 教训）；`dotnet test --arch x64`。
- 前端：无测试运行器，靠 `vue-tsc` 过滤本模块文件 + `vite build` 兜底 + 真机走查。
- 真机：worktree 内起 dev server（注意与主树 9000/9001 端口冲突，必要时换端口）；读写共享远程库时只采样、隔离测试数据。

## 5. 风险

- 菜单种子可能未进目标库 → 侧栏无宿舍入口（Phase 1 末核验，必要时走 DbMigration/重放种子）。
- 共享远程库被并发会话清空风险（记忆 org192）→ 测试数据隔离、操作前确认组织。
- 真机端口冲突 → 用独立端口或先停主树 dev。
- 合并回 master 时机由用户定（记忆：不擅自 push / 开分支已获批）。
