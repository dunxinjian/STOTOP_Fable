# 损益类禁录期初（P2）— 设计

- 日期：2026-06-19
- 范围：期初余额录入禁止损益类科目录入非 0 期初（前端置灰 + 后端守卫）。
- 优先级：P2（会计正确性，避免损益录期初造成的无意义/凑平假象）
- 工作区：worktree `finance-p2`（分支 `worktree-finance-p2`）

## 1. 问题与会计依据

期初表对该账套所有末级科目无差别开放录入（`GetInitialBalancesAsync` 只过滤 `FEnableStatus==1`，不按类别排除），含损益类（5 开头：营业收入/营业成本/营业税金及附加/期间费用/其他收益/其他损失/所得税费用/以前年度损益调整）。

会计上**损益类年初余额恒为 0**（损益类每年结转清零、不结转余额到下年）。给损益科目录期初借贷余额无会计意义，还会让试算平衡"凑平"出假象。且 `InitialBalanceDto`（`src/STOTOP.Module.Finance/Dtos/AccountDto.cs:56-64`）无 `Category` 字段，前端无依据置灰损益行。

## 2. 修复设计（前端置灰 + 后端守卫双层）

### 2.1 复用单一真源
损益判定复用 `STOTOP.Module.Finance.Constants.FinAccountCategory.IsProfitLoss(category)`（损益取数修复那轮建立的单一真源，含 8 子类 + "损益"）。

### 2.2 判定口径
- 后端守卫用科目实体 `FCategory` + `FinAccountCategory.IsProfitLoss`（权威）。
- 前端置灰用 `record.accountCode` 5 开头判定（损益科目编码铁定以 5 开头，与 `IsProfitLoss` 对损益等价、最简，**无需给 `InitialBalanceDto` 加字段**——避免无消费者的冗余字段）。

### 2.3 后端守卫（权威，防绕过）
`SaveInitialBalancesAsync`（`AccountService.cs:366-429`，现已含上一轮"全量快照试算平衡"校验）：
- 现有 `accounts` 查询补 `FCategory`（现 `Select(a => new { a.FID, a.FParentId })` → 增 `a.FCategory`）。
- 在"归属/仅末级"校验之后、试算平衡校验之前，加损益守卫：对每个提交 item，若其科目 `IsProfitLoss(category)` 且 `DebitBalance != 0 || CreditBalance != 0`，抛 `InvalidOperationException($"损益类科目（{code}）不可录入期初余额")`。
- 损益科目提交 0 期初不报错（允许前端全量提交含损益的 0 值行）。

### 2.4 前端置灰
期初表（`web/src/views/finance/AccountManage.vue` 期初 `a-input-number`，约 145-167 行）：
- 借/贷 `a-input-number` 的 `:disabled` 在原条件（按 `balanceDirection` + `isLeaf`）基础上，叠加"损益行禁用"：损益行（`record.category` 属损益）借贷都 disabled、值锁 0。
- 损益判定前端用 `record.accountCode` 5 开头（损益科目编码必以 5 开头）。
- 提示：工具栏 `initial-tip` 或损益行加说明"损益类科目不录期初"。

## 3. 测试

- **后端集成**（InMemory + 真 Repository，参照 `AccountServiceInitialBalanceTests`）：
  - seed 损益末级科目（如 5001 营业收入，FCategory="营业收入"）+ 资产/权益末级科目；
  - 提交损益科目非 0 期初 → `SaveInitialBalancesAsync` 抛 `InvalidOperationException`（消息含"损益"）、不落库；
  - 损益科目 0 期初 + 资产/权益借贷平衡 → 正常落库（损益不参与、平衡通过）。
- 前端：无测试运行器，`vite build` + 手动（损益行置灰、录入被拦）。

## 4. 验证方式

- `dotnet build` Finance 0 错误。
- `dotnet test tests/STOTOP.Module.Finance.Tests --arch x64`：新增集成测试红→绿，既有不回归。
- 前端 `vite build` 通过。

## 5. 非目标（YAGNI）

- 不做"本年累计发生额"录入（年中启用损益累计场景，另立）。
- 不改试算平衡校验（损益置 0 天然不影响 Σ借==Σ贷）。
- 不动其它类别（资产/负债/权益/成本照常可录期初）。
