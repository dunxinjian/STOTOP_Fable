# 辅助核算入口（P2）— 设计

- 日期：2026-06-19
- 范围：科目表辅助核算列加"设置"入口，跳转辅助核算维护页，消除"找不到入口"断裂。
- 优先级：P2（可发现性）
- 工作区：worktree `finance-p2c`

## 1. 问题

科目表能看「辅助核算」列（`formatAuxiliary(record.auxiliary)`），但弹窗内不能改（仅透传），真正维护在 `AuxiliarySetting`（路由 `finance/auxiliary-setting`），科目页**无任何文字/链接指向它**——看得到、改不了、找不到入口。

## 2. 修复设计（纯前端简单跳转）

`AccountManage.vue`：
- `import { useRouter } from 'vue-router'`，`const router = useRouter()`。
- 辅助核算列单元格（`#bodyCell` 的 `auxiliary` 分支）在 `formatAuxiliary` 显示后追加一个弱化的「设置」链接/图标，`@click` 调 `router.push({ name: 'AuxiliarySetting' })` 跳转辅助核算维护页。

`AuxiliarySetting` 不动（结构复杂、预定位语义不直接，预定位另立优化）。

## 3. 测试

- 无后端改动 → 无新测试。
- 前端 `npx vite build` 通过 + 手动（点链接跳转到辅助核算设置页）。

## 4. 非目标（YAGNI）

- 不带科目预定位（`AuxiliarySetting` 读 query 预选——结构复杂，另立）。
- 不在科目弹窗内嵌辅助核算编辑（需 SubjectEdit/AuxiliaryEdit 权限协调）。
