# 工作台：去掉列表卡片内联操作按钮，操作收敛到右栏详情 — 设计

- **日期**：2026-06-18
- **范围**：工作台中栏工作项卡片（`WorkItemCard.vue`）与右栏详情（`WorkHubDetail.vue`）。
- **背景**：卡片点击已在右栏 `WorkHubDetail` 打开详情、详情底部已有操作按钮，故列表卡片上的「进入处理」等内联主操作按钮冗余，且占用卡片高度。把操作收敛到右栏，卡片回归"点击即预览"。

## 决策（已与用户确认）
- 审批/任务类卡片**彻底去掉**内联操作（主按钮 + 「...」次要菜单）；**数据导入类卡片保留**其专有按钮（详情面板未覆盖这些专门跳转）。
- 卡片专有的「稍后处理」（defer）随之从卡片移除，**改加到右栏详情**底部操作条。
- 多选批量操作栏、通知详情、数据导入卡保持不动。

## A. WorkItemCard.vue

分支现状（模板互斥链）：`v-if="source==='oa'"`（OA 增强）/ `v-else-if="isDataImport && diSubType"`（数据导入各 subType）/ `v-else`（默认：CardFlow/质量/任务/合同等，Task 6 已并为 `.card-line-row` 合并行）。

1. **默认分支（`v-else`）**：删除 `.card-line-row` 内的 `<div class="card-actions" @click.stop>…</div>`（含 `primaryActions` 主按钮循环 + `secondaryActions` 的 `<a-dropdown>`「...」菜单）。保留 `.card-line`（链接·摘要）与 `.deadline`。结果：默认卡无任何内联操作，点击整卡 → 右栏预览。
2. **OA 分支（`v-if="source==='oa'"`）**：删除该分支末尾整个 `<div class="card-footer">…</div>`（`card-meta` 空占位 + `card-actions`：主按钮 + 「...」+ 撤回按钮）。保留 `oa-node-row` / `card-summary` / `oa-enhanced`（节点、摘要、截止、审批模式）。
3. **数据导入分支（各 subType）**：**不改**。其 `card-footer`/`card-actions`/专有按钮（查看详情 → `/cardflow/upload-center`、查看分类结果、开始处理、`error_dispatched` 的 primary/secondary、unknown subType 的默认 footer）全部保留。
4. **脚本清理（"改到哪清到哪"）**：
   - `handleWithdraw`：仅 OA 撤回按钮调用，移除该按钮后变为未用 → 删除该函数；同时删除随之未用的 import（`RollbackOutlined`；以及若 `Modal`、`message` 在本文件不再被其它处使用，一并删除——实现时以全文搜索结果为准，只删确实未用的）。
   - **保留** `primaryActions` / `secondaryActions` / `handleAction` / `handleDefer`（数据导入分支的 `error_dispatched`、unknown-subtype 仍在用）。
   - **保留** `.card-actions` / `.card-footer` / `.card-meta` / `.more-actions-btn` 等样式（数据导入分支仍用）。

## B. WorkHubDetail.vue

1. 新增 `handleDefer()`：
   ```ts
   function handleDefer() {
     if (!workItem.value) return
     hub.deferItem(workItem.value.id)
     emit('close')
   }
   ```
   （`hub` 已由 `const hub = useWorkHub()` 提供；`deferItem` 已在 useWorkHub 暴露；`emit('close')` 已定义，父组件 `index.vue` 据此清空右栏选中。）
2. 底部操作条 `.wi-actions`：
   - 渲染条件由 `v-if="workItem.actions && workItem.actions.length > 0"` 改为**工作项一律渲染**（即便无 actions 也要能"稍后处理"）。
   - 在操作按钮**之前**加一个左侧「稍后处理」按钮：`<a-button type="text" size="small" @click="handleDefer">`（图标 `ClockCircleOutlined`，已 import）。主操作按钮 `wi-action-btn` 仍在右侧（`flex:1` 撑开），稍后处理按钮 shrink 居左。
   - 通知详情分支（notification）不涉及（`wi-actions` 在 workitem 分支内）。

## 不动
多选/批量操作（`WorkHubCenter` 的 batch-action-bar 与 `useWorkHub` 多选逻辑）、通知详情、数据导入卡按钮、`CardActivityTimeline`。

## 测试与验证
- 无前端测试运行器（见 [[frontend-typecheck-baseline-red]]）：`cd web && npm run build` 编译门禁 + `npm run lint:style`（不新增裸 hex）。
- 手动（preview，用户登录态）：
  1. CardFlow/质量/任务/合同卡 → 无内联按钮、无「...」；点整卡 → 右栏详情；右栏底部有 主操作 + 「稍后处理」。
  2. 「稍后处理」→ 该项进入中栏"稍后处理"折叠区、右栏关闭。
  3. 数据导入卡 → 专有按钮仍在、仍可用。
  4. 多选模式仍可勾选 + 批量批准/驳回。
  5. 卡片明显更矮（默认卡少了操作行）。

## 风险
- OA 分支属待清理（[[oa-abolished-by-cardflow]]）；本次仅去其内联操作并清理随之而来的死代码，不动其它 OA 渲染。
- `wi-actions` 改为常显：若某工作项确无任何 action 也仅显示「稍后处理」，符合预期（不留空条——始终至少有稍后处理）。
