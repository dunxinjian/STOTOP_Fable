# 期初试算平衡校验（P1-3）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 期初余额录入增加「借方合计 = 贷方合计」试算平衡校验，后端为权威拦截不平保存，前端即时显示差额并禁用保存。

**Architecture:** 后端新建纯函数校验器 `InitialBalanceValidator`（全量快照：每个末级科目期初取「提交值→库中现值→0」），接入 `SaveInitialBalancesAsync`，并补 `AccountController` 异常转译；前端 `AccountManage.vue` 加 computed 差额 + summary 红字 + 禁用保存。无数据迁移。

**Tech Stack:** .NET 10 / EF Core（InMemory 测试）/ xUnit；Vue 3 + ant-design-vue。

> **背景（实现者必读）**：期初录入当前无借贷平衡校验，可保存「借 100 万 / 贷 80 万」的不平期初，污染整个账套后续期间滚动。`SaveInitialBalancesAsync`（`src/STOTOP.Module.Finance/Services/AccountService.cs:366-429`）已校验科目归属与「仅末级可录」，唯独漏了借贷平衡。详见 `docs/superpowers/specs/2026-06-19-期初试算平衡-design.md`。
>
> **测试约定**：运行测试加 `--arch x64`。后端集成测试用 InMemory：`TestDbContextFactory.Create(name)` 造 `STOTOPDbContext`，`new Repository<T>(db)`（单参构造）造仓储；`AccountService` 构造 5 参数 `(accountRepo, balanceRepo, voucherEntryRepo, changeTracking, httpContextAccessor)`，而 `SaveInitialBalancesAsync` 只用前两个，其余传 `null!`。若 dll 被开发服务占用致 build 失败，回报 BLOCKED。

```csharp
namespace STOTOP.Module.Finance.Services;

/// <summary>
/// 期初余额试算平衡校验（纯函数，便于单测）。
/// 全量快照语义：对账套全部末级科目，期初借/贷取「本次提交值 → 库中现值 → 0」，比较 Σ借 与 Σ贷。
/// </summary>
public static class InitialBalanceValidator
{
    /// <summary>金额 2 位小数容差。</summary>
    public const decimal Tolerance = 0.005m;

    /// <summary>
    /// 计算「提交覆盖后」全部末级科目期初的借方合计、贷方合计。
    /// 每个末级科目取 submitted 命中值，否则 existing 命中值，否则 (0,0)。
    /// </summary>
    public static (decimal Debit, decimal Credit) ComputeTotals(
        IEnumerable<long> leafAccountIds,
        IReadOnlyDictionary<long, (decimal Debit, decimal Credit)> existing,
        IReadOnlyDictionary<long, (decimal Debit, decimal Credit)> submitted)
    {
        decimal totalDebit = 0m, totalCredit = 0m;
        foreach (var id in leafAccountIds)
        {
            (decimal Debit, decimal Credit) v =
                submitted.TryGetValue(id, out var s) ? s
                : existing.TryGetValue(id, out var e) ? e
                : (0m, 0m);
            totalDebit += v.Debit;
            totalCredit += v.Credit;
        }
        return (totalDebit, totalCredit);
    }

    /// <summary>差额在容差内即视为平衡。</summary>
    public static bool IsBalanced(decimal difference) => Math.Abs(difference) < Tolerance;
}
```
