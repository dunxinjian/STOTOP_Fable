# 快递行业模板空数据源修复（P1-4）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为「快递行业标准模板」(FID3) 补明细科目（账套2快照，排除 1002/1012 私有银行账户子科目），修复建账选快递模板必报错。

**Architecture:** 新增 FinanceSeeder `MigrateV11` 用 `INSERT...SELECT` 从账套2科目快照成 FID3 模板明细（模板项 FID 复用科目 FID 保父子）；顺带 `sourceId -2→2`；ApplyTemplate 机制由集成测试锁定，迁移 SQL 靠真库验证。

**Tech Stack:** .NET 10 / EF Core / xUnit；SQL Server 迁移（`ExecSql`）。

> **背景（实现者必读）**：FID3「快递行业标准模板」无明细 → 建账选它报「模板中没有科目数据」（`AccountTemplateService.cs:288`）。`ApplyTemplateAsync`（`:274-341`）按 FLevel 排序父先建、用 `idMapping[模板项FID→新科目FID]` 还原父子；模板项 `F父ID` 引用父模板项 FID。详见 spec `docs/superpowers/specs/2026-06-19-快递模板空数据源-design.md`。
>
> **迁移机制**：`FinanceSeeder.Migrate` 用 `MigrationRunner.RunMigrations` 跑 `MigrationStep` 列表，每个 `MigrateVxx(STOTOPDbContext ctx)` 在单事务内执行，用 `ExecSql(ctx, sql)` 跑原生 SQL（参照 `MigrateV10`，`:2190`）。`FIN科目模板_明细` 是基线表，由 `DatabaseSeederAdapter` 自动建缺失表，无需兜底 CREATE。
>
> **测试约定**：`--arch x64`。集成测试用 `TestDbContextFactory.Create(name)` + `new Repository<T>(db)`。`ApplyTemplate` 用事务，EF InMemory 对事务是 no-op + warning（不抛）；若测试因事务报错，在该测试构造 db 时忽略 `InMemoryEventId.TransactionIgnoredWarning`（实现者按报错处理）。

```csharp
    private static void MigrateV11(STOTOPDbContext ctx)
    {
        if (!SeederHelper.IsSqlServer(ctx)) return;

        // FID3「快递行业标准模板」明细 = 账套2(太仓美申)科目快照。
        // 模板项 FID 复用科目 FID、F父ID 直接复制 → ApplyTemplate 的 idMapping 还原父子。
        // 排除 1002/1012 下账套私有银行账户子科目；二者本身设为叶子。
        ExecSql(ctx, @"DELETE FROM [FIN科目模板_明细] WHERE [F模板ID]=3;");

        ExecSql(ctx, @"
            SET IDENTITY_INSERT [FIN科目模板_明细] ON;
            INSERT INTO [FIN科目模板_明细]
              ([FID],[F模板ID],[F编码],[F名称],[F类别],[F余额方向],[F级次],[F父ID],[F是否末级],[F辅助核算],[F外币],[F计算单位],[F排序])
            SELECT [FID], 3, [F编码], [F名称], [F类别], [F余额方向], [F级次], [F父ID],
                   CASE WHEN [F编码] IN (N'1002', N'1012') THEN 1 ELSE [F是否末级] END,
                   [F辅助核算], [F外币], [F计算单位], 0
            FROM [FIN科目]
            WHERE [F账套ID]=2
              AND NOT ([F编码] LIKE N'1002%' AND LEN([F编码]) > 4)
              AND NOT ([F编码] LIKE N'1012%' AND LEN([F编码]) > 4);
            SET IDENTITY_INSERT [FIN科目模板_明细] OFF;");

        // 模板说明与实际数据源对齐
        ExecSql(ctx, @"UPDATE [FIN科目模板]
            SET [F说明]=N'源自太仓美申账套快照（排除各账套私有银行账户子科目）'
            WHERE [FID]=3;");
    }
```

> 注意：若真库跑迁移报「表 FIN科目模板_明细 没有标识列」之类错误，说明 FID 非 IDENTITY，去掉两行 `SET IDENTITY_INSERT ... ON/OFF` 即可（INSERT 显式 FID 列照常）。先按上面写。
