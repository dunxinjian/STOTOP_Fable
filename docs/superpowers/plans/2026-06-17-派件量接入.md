# 派件量接入 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让派件量经 CardFlow 暂存框架导入 `STG申通派件日明细`，并让阿米巴公共费分摊的 deliver 件量从该表取数，根治 deliver=0 / scope 不感知 / 跨月周近似 三问题。

**Architecture:** Part A（CardFlow 导入）新增 EF 实体型 STG 表 + `MigrateV23` 迁移（建表/导入规则/流程三件套）+ StagingService/Revoke 接表。Part B（Finance 取数）新增 `AggregateDeliverVolume` 原始 SQL，改 `ComputeVolumeBasis`/`ApplyCommonCostAllocation` 为 deliver 双注入，在 S2/S4 两调用点各算两次取数。两半经 `STG申通派件日明细` 表名解耦。

**Tech Stack:** C# / .NET（EF Core SQL Server）、xUnit（含 SkippableFact SQL Server 冒烟）、Vue 3 + TypeScript（可选前端 tab）。

**关键依据：** 设计 spec `docs/superpowers/specs/2026-06-17-派件量接入-design.md`（已评审修订，确认 24 项）。三决策：去重方案 B（4 字段唯一索引 + 关插件去重，重复导入整批失败）/ 展示改源 v1 延后 / 账套按组织。

**全局命令（Windows / PowerShell）：**
- Build：`dotnet build src/STOTOP.sln`
- 测试（必须 `--arch x64`，避免 x86 testhost WSAENOBUFS 中止）：`dotnet test tests/<项目>/<项目>.csproj --arch x64 --filter "<过滤>"`
- SQL Server 冒烟需设环境变量 `STOTOP_TEST_CONNECTION`，未设则 `[SkippableFact]` 自动跳过。

**三处列名逐字一致铁律：** 实体 Config / `MigrateV23` DDL / 规则 `columnMapping.dbColumn` 三处列名必须完全相同（含 `F基础派费和时效拦截弃件付费件量合计` 这类长名、`F综合KPI奖励派费件量` 大写 KPI）。本计划 Task A1 写出的列名即唯一真源，A2 严格照抄。

```csharp
internal void ApplyCommonCostAllocation(
    Dictionary<long, decimal> scopedMatched,
    List<DataPoint> scopedPoints,
    List<DataPoint> fullPoints,
    List<FinAmoebaPLItem> plItems,
    List<FinAmoebaPLItem> indicatorItems,
    List<FinAmoebaPLItem> allItems,
    List<string> warnings,
    long scopedDeliver,
    long fullDeliver)
{
    var commonLeaves = plItems.Where(IsCommonCostLeaf).ToList();
    if (commonLeaves.Count == 0) return;

    var (fullMatched, _) = MatchDataPointsToPLItems(fullPoints, plItems);
    if (indicatorItems.Count > 0)
    {
        var (fullInd, _) = MatchDataPointsToPLItems(fullPoints, indicatorItems);
        foreach (var kv in fullInd) fullMatched[kv.Key] = kv.Value;
    }

    var fullTotals = new Dictionary<long, decimal>();
    foreach (var leaf in commonLeaves)
        if (fullMatched.TryGetValue(leaf.FID, out var t)) fullTotals[leaf.FID] = t;

    var (sendId, _) = ResolveVolumeIndicatorIds(allItems);   // deliverId 不再用于取数
    var scopeVol = ComputeVolumeBasis(scopedPoints, scopedMatched, sendId, scopedDeliver);
    var fullVol = ComputeVolumeBasis(fullPoints, fullMatched, sendId, fullDeliver);

    var alloc = _allocEngine.Allocate(commonLeaves, fullTotals, scopeVol, fullVol, out var w);
    warnings.AddRange(w);

    foreach (var leaf in commonLeaves)
    {
        if (alloc.TryGetValue(leaf.FID, out var a)) scopedMatched[leaf.FID] = a;
        else scopedMatched.Remove(leaf.FID);
    }
}
```

```csharp
private sealed class DeliverSumRow { public long Value { get; set; } }

/// <summary>
/// 从 STG申通派件日明细 聚合 deliver 件量（设计 spec §B1）。平行 AggregateBillingRevenue：
/// 参数化 SQL、半开区间、按组织(不按账套)；过滤列用系统列 [F归属网点编号]（与 billing dp.SiteCode/scope.Outlets 同源）。
/// </summary>
private async Task<long> AggregateDeliverVolume(
    DateTime startDate, DateTime endDate, long orgId, AmoebaReportScope? scope)
{
    var outlets = scope?.Outlets?.Where(s => !string.IsNullOrEmpty(s)).Distinct().ToList();

    var sql = @"SELECT ISNULL(SUM([F基础派费收费件量]), 0) AS [Value]
                FROM [STG申通派件日明细]
                WHERE [F结算日期] >= {0} AND [F结算日期] < {1}
                    AND [FOrgId] = {2} AND [FIsRevoked] = 0";
    var parameters = new List<object> { startDate, endDate.AddDays(1), orgId };  // 半开 [起, 末日+1)

    if (outlets is { Count: > 0 })
    {
        var inClause = string.Join(",", outlets.Select((_, i) => $"{{{i + 3}}}"));
        sql += $" AND [F归属网点编号] IN ({inClause})";
        parameters.AddRange(outlets.Cast<object>());
    }

    var row = await _dbContext.Database
        .SqlQueryRaw<DeliverSumRow>(sql, parameters.ToArray())
        .FirstAsync();
    return row.Value;
}
```

> 若项目里 `SqlQueryRaw<long>(... AS [Value])` 已被其它标量查询验证可用（如 Smoke 测试用 `SqlQueryRaw<int>`），可省去 `DeliverSumRow` 直接 `SqlQueryRaw<long>`。二选一，与既有惯例对齐。
