# 登录精简 Implementation Plan（绿色项②）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实施。步骤用 `- [ ]` 复选框跟踪。

**Goal:** 去掉登录成功后那段"4 步伪加载 + 最小 2300ms"的剧场式过渡屏（改为登录真实工作完成即跳转，期间复用现有 loading 态）；把多组织选择从模态弹窗改为登录页内联面板。

**Architecture:** `login/index.vue` 的 `runPostLoginTransition` 删掉 `Promise.all([loadOrgsAndRedirect(), wait(2300)])` 与 4 步定时器播放，改为直接 `await loadOrgsAndRedirect()`；多组织无主组织分支不再开 `OrgSelectModal`，改在登录右栏内联渲染一个组织选择面板，选中走原 `handleOrgSelected`。单组织/有主组织自动跳过的现状行为保留不动。

**Tech Stack:** Vue 3 + Pinia + Ant Design Vue。**前端无单测运行器**，验证靠 `cd web && npm run build` + 预览；`vue-tsc -b` 基线红不当门禁；样式禁裸 hex（用 `var(--color-*)` 令牌，过 stylelint）。**不在本计划**：扫码 SSO 扶正为主（属另案 UI 重排）、组织选择板带实时状态（愿景版/红色项）。

把 `runPostLoginTransition`(L375-401) 整体替换为（**保留**原函数里登录后 `fetchSecurityConfig` / `initIdleDetection` / `router.push` 这套真实收尾，仅去掉 `Promise.all` 的 `wait(2300)` 与步骤播放；变量/函数名以文件实际为准）：

```ts
async function runPostLoginTransition() {
  loading.value = true
  try {
    const result = await loadOrgsAndRedirect()
    if (result?.type === 'redirect') {
      await fetchSecurityConfig()
      initIdleDetection()
      router.push(result.redirect ?? '/')
    }
    // result.type === 'org-selection'：loadOrgsAndRedirect 已置内联组织选择态（见 Task 2），留在本页
  } finally {
    loading.value = false
  }
}
```
