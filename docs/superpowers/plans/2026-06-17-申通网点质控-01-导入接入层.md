# 申通网点质控 · 导入接入层 实现计划（Plan 1 / 3）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让用户一次拖入整批申通质控 Excel，按表头内容自动路由，全部入对应 `STG申通_*` 暂存表（约 27 张），带去重/批次/组织隔离，未匹配文件给出「待认领」清单。

**Architecture:** 复用 CardFlow 暂存导入框架——单源导入零新增代码（实体 + EF 配置 + CREATE 迁移 + `CfPluginRule` 种子 + 流程/版本/节点种子）。仅三处框架增强：① 解析按文件头魔数判别 xls/xlsx（兼容错命名文件）；② 解析支持指定 sheet（兼容 3-sheet 的物流信息指数）；③ 新增「多文件内容路由」批量上传入口（前端 + 后端），复用已实现的 `BatchTriggerService.MatchFlowDefinitionsAsync` 做表头指纹匹配。

**Tech Stack:** C# / .NET（`STOTOP.Module.CardFlow`）、EF Core（`IEntityTypeConfiguration` 自动扫描注册）、MiniExcel(.xlsx)/NPOI(.xls)、SQL Server、Vue 3 + ant-design-vue（上传页）。测试：xUnit（`dotnet test --arch x64`，见 [[dotnet-test-x64-socket]]）。

**关联：** spec `docs/superpowers/specs/2026-06-17-申通网点质控看板-design.md`（§5 接入层）；框架契约见该 spec 与 CardFlow 现有代码；约定见 [[stotop-cardflow-staging-import]]。

- [ ] **Step 3: 实现魔数嗅探，按内容选引擎**

在 `ExcelParserService` 加私有方法，并让 `ParseAsync` 的非 csv 分支改为按嗅探结果选 `.xlsx`/`.xls` 路径（而非扩展名）：

```csharp
private static async Task<string> SniffExcelFormatAsync(Stream s, string fallbackExt, CancellationToken ct)
{
    if (!s.CanSeek) return fallbackExt; // 不可定位则退回扩展名
    var head = new byte[8];
    var pos = s.Position;
    var n = await s.ReadAsync(head.AsMemory(0, 8), ct);
    s.Position = pos;
    if (n >= 4 && head[0] == 0x50 && head[1] == 0x4B && head[2] == 0x03 && head[3] == 0x04)
        return ".xlsx"; // ZIP/OOXML
    if (n >= 8 && head[0] == 0xD0 && head[1] == 0xCF && head[2] == 0x11 && head[3] == 0xE0)
        return ".xls";  // OLE2/BIFF
    return fallbackExt;
}
```
并在 `ParseAsync` 非 csv 分支：
```csharp
var realExt = await SniffExcelFormatAsync(fileStream, extension, ct);
await ParseExcelAsync(fileStream, realExt, headerRow, dataStartRow, batchSize, batchCallback, ct);
```
（若上传入口先把流复制到 `MemoryStream` 则一定可 seek；不可 seek 时退回扩展名，行为不回退。）

- [ ] **Step 5: 加 `upload-auto` 控制器方法（多文件 + 内容路由 + 待认领）**

在 `CardFlowBatchController` 注入 `ExcelParserService` 与 `BatchTriggerService`（确认其可注入），新增：
```csharp
/// <summary>多文件上传：逐文件按表头内容路由到流程，未匹配进待认领</summary>
[HttpPost("upload-auto")]
[Consumes("multipart/form-data")]
public async Task<ApiResult<object>> UploadAuto(IFormFileCollection files)
{
    var orgId = GetOrgId();
    if (orgId == 0) return ApiResult<object>.Fail("当前无组织上下文", 401);
    if (files == null || files.Count == 0) return ApiResult<object>.Fail("请上传文件");

    var uploadDir = Path.Combine(_env.ContentRootPath, "uploads", "cardflow");
    Directory.CreateDirectory(uploadDir);

    var routed = new List<object>();
    var unmatched = new List<object>();
    var ambiguous = new List<object>();

    foreach (var file in files)
    {
        if (file.Length == 0) continue;
        var safeName = $"{DateTime.Now:yyyyMMddHHmmssfff}_{Guid.NewGuid():N}{Path.GetExtension(file.FileName)}";
        var savedPath = Path.Combine(uploadDir, safeName);
        await using (var fs = global::System.IO.File.Create(savedPath)) await file.CopyToAsync(fs);

        IReadOnlyList<string> cols;
        await using (var rs = global::System.IO.File.OpenRead(savedPath))
            cols = await _parser.ReadHeadersAsync(rs, file.FileName, headerRow: 1);

        var matches = await _triggerService.MatchFlowDefinitionsAsync(cols, file.FileName, orgId);
        if (matches.Count == 0) { unmatched.Add(new { file.FileName, columns = cols }); continue; }
        if (matches.Count > 1) { ambiguous.Add(new { file.FileName, candidates = matches.Select(m => m.FlowDefinitionId) }); continue; }

        var batchId = await _triggerService.TriggerByFileUploadAsync(
            matches[0].FlowDefinitionId, orgId, GetUserId(), savedPath, new Dictionary<string, string>());
        routed.Add(new { file.FileName, batchId, flowDefinitionId = matches[0].FlowDefinitionId });
    }

    return ApiResult<object>.Success(new { routed, unmatched, ambiguous },
        $"已路由 {routed.Count} 个；待认领 {unmatched.Count} 个；多义 {ambiguous.Count} 个");
}
```
（`MatchResult` 的属性名以 Task 0 / `BatchTriggerService` 实际为准；多义 `ambiguous` 一期作提示，由用户后续手工指派——前端在 Task A4 呈现。）

## Phase B：27 个 STG 源（纯数据，四件套）

**机制（重要）：** 单源导入**不需要新增任何 C# 业务代码**——`ExcelInputPlugin` 配置驱动。每个源做四件事：(a) 实体类，(b) EF 配置（`.ToTable("STG申通_xxx")`），(c) `CardFlowSeeder` 加一个 `MigrateVN`（建表 + 插 `CfPluginRule` + 插 `CF卡片流程`/版本/节点），(d) 在 `Migrate()` 的 `steps` 登记该 `MigrateVN`。

**列与映射的真源 = 实际 Excel 文件**：每个源的完整列、`columnMapping`（excelColumn→dbColumn）从对应文件首行读取（spec §5.2 已列出文件↔表↔列指纹；文件路径见附录）。`columnIdentifier` 用附录给出的「必含列」（路由指纹）。

**ID 段约定：** 现有规则用到 3001/3002…、流程 22xx、节点 50xx。本计划质控源统一用新段避免冲突：`CfPluginRule.FID` = 3101–3127；`CF卡片流程.FID` = 2301–2327；`CF流程版本.FID` = 2301–2327；首节点 `FID` = 5101–5127。Task B0 锁定分配表，后续逐源取用。

**去重规则（重要，B0 评审确定）：** `crossBatchDedupFields` 与 `keyFields` 统一用 **≤2 个字段**的「事件身份」——问题件清单用 `运单号`(+`问题类型`/`工单号` 等)，忽略导出日期；汇总源用 `实体(网点编码/工号/网点编号)`+`统计日期/周期`。唯一索引 = 去重键(DB 列名) + `FOrgId`、带 `[FIsRevoked]=0` 过滤，**与 `crossBatchDedupFields` 一一对应**。原因：`ExcelInputPlugin.QueryExistingByFieldsAsync` 对 **≥3 个 dedup 字段失效**（JOIN 第三列恒 NULL → 去重不生效且撞唯一索引时丢整批）；该插件 bug 已另案修复（task_bce686c8），在其修好前所有源 dedup 一律 ≤2 字段。每源接入后须验证「同文件导入两次行数不变」的幂等性。
