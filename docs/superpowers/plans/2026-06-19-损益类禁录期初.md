# 损益类禁录期初（P2）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 期初余额录入禁止损益类科目录入非 0 期初（后端守卫 + 前端置灰），复用 `FinAccountCategory.IsProfitLoss` 单一真源。

**Architecture:** `SaveInitialBalancesAsync` 在归属/末级校验时加损益守卫（损益类非 0 抛异常，权威）；前端期初表对损益行（`accountCode` 5 开头）借贷 input 置灰锁 0。判定不依赖新增 DTO 字段，无数据迁移。

**Tech Stack:** .NET 10 / EF Core（InMemory 测试）/ xUnit；Vue 3 + ant-design-vue。工作区：worktree `finance-p2`。

> **背景**：期初表对所有末级科目开放录入（含损益类 5 开头），损益类年初余额会计上恒 0、录期初无意义。详见 spec `docs/superpowers/specs/2026-06-19-损益类禁录期初-design.md`。`SaveInitialBalancesAsync`（`src/STOTOP.Module.Finance/Services/AccountService.cs:367`）现含「归属/末级校验 → 试算平衡校验 → upsert」三段；本次在归属/末级校验段加损益守卫。`AccountService` 已 `using STOTOP.Module.Finance.Constants`，`FinAccountCategory.IsProfitLoss(string?)` 可直接用。
>
> **判定口径**：后端用实体 `FCategory` + `IsProfitLoss`；前端用 `accountCode` 5 开头（损益编码必 5 开头，最简，无需给 DTO 加字段）。
>
> **测试约定**：`--arch x64`；集成测试参照 `AccountServiceInitialBalanceTests`（`CreateService(db)` = `new AccountService(repo, repo, null!, null!, null!)`）。命令在 worktree 根目录执行。

改为（增损益守卫分支）：

```csharp
        foreach (var item in request.Items)
        {
            if (!accountIds.Contains(item.AccountId))
            {
                throw new InvalidOperationException($"科目 {item.AccountId} 不属于当前账套");
            }
            if (parentIds.Contains(item.AccountId))
            {
                throw new InvalidOperationException("仅末级科目可录入期初余额");
            }
            // 损益类科目年初余额恒 0，禁止录入非 0 期初（0 期初允许跳过）
            if ((item.DebitBalance != 0 || item.CreditBalance != 0)
                && accountInfoMap.TryGetValue(item.AccountId, out var info)
                && FinAccountCategory.IsProfitLoss(info.FCategory))
            {
                throw new InvalidOperationException($"损益类科目（{info.FCode}）不可录入期初余额");
            }
        }
```
