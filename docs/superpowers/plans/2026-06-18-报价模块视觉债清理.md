# 报价模块视觉债清理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把报价模块 16 个文件里约 87 处裸 hex 颜色字面量换成新令牌，清零旧蓝、扁平渐变，对齐「中性墨 + 橙点缀」，不动布局/交互/计费。

**Architecture:** 先加 2 个新令牌（Task 0，后续任务依赖），再逐文件做查找替换（Task 1-8，文件互不依赖、可并行），最后整体验证（Task 9）。统一用 `var(--token)` 落地（`<style>`/内联 `style`/`h()` render 字符串三处都用 `var()`）。

**Tech Stack:** Vue 3 SFC + SCSS、Ant Design Vue、设计令牌（`theme.ts applyDesignTokensCSS()` 注入 + `variables.scss` 桥接）。无前端测试运行器——验证靠 `npm run build`（vite）+ `npx stylelint` + `rg` 复扫裸 hex 清零 + 用户 live HMR 目视。

**Spec:** `docs/superpowers/specs/2026-06-18-报价模块视觉债清理-design.md`（§3 映射规范表、§4 D1-D5 决策为本计划权威依据）。

**通用约定（每个文件任务都适用）：**
- 按 spec §3 规范表替换中性 hex；按 §4 D1-D5 处理旧蓝/渐变/色块白字/焦点环特例。
- 替换值一律 `var(--token)`，不新增 SCSS `$` 写法、不混用、不为省事把内联样式抽成 class（结构不动）。
- 不碰模板结构、事件、计费、数据、布局间距；只改颜色值。
- 每文件完成后：`npx stylelint <file>` = 0；`rg "#[0-9a-fA-F]{3,6}" <file>` 复扫（组件文件应 **0 命中**——所有 hex 都已变 `var()`；rgba(0,0,0,*) 阴影非 hex、不在范围）。
