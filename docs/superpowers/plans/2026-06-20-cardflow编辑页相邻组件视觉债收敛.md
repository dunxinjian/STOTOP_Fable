# CardFlow 编辑页相邻组件视觉债收敛 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把编辑页内联渲染的 6 个相邻组件（~190 处裸 hex）对齐全局令牌体系——CardComponentRenderer 橙化+去绿，两套分类色提升为 `--cf-*` 命名令牌，其余中性/语义换源。

**Architecture:** 先在令牌真源（`theme.ts` 的 `applyDesignTokensCSS()` + `TOKENS.md`）新增 15 个静态分类令牌（性质同 `--biz-*`/`--avatar-palette-*`，**不加 `variables.scss` `$` 桥接**，镜像 biz 现状）；再逐文件按「值→令牌字典」替换裸 hex 为 `var()` / `color-mix()`。纯前端、无后端、无单测。

**Tech Stack:** Vue 3 SFC（scoped SCSS + 模板内联 `style` + JS `tone` 字符串）、CSS 自定义属性、`color-mix()`、ripgrep + stylelint + vite build 做门禁。

**设计依据：** [2026-06-20-cardflow编辑页相邻组件视觉债收敛-design.md](../specs/2026-06-20-cardflow编辑页相邻组件视觉债收敛-design.md)；中性/语义口径复用上一轮 [2026-06-19 spec §5](../specs/2026-06-19-cardflow流程编辑器视觉债收敛-design.md)。

## 通用执行注意（每个替换任务都适用）

1. **子串安全**：`Edit` 是字面子串替换。短 hex（`#fff`/`#eee`/`#666`/`#999`/`#333`/`#bbb`）替换时**必须带分隔上下文**（如 `: #fff;`、`#fff,`、`#fff)`、`solid #fff`），或先替换更长的同前缀值。本任务各文件 grep 已确认无 `#ffffff` 等长同前缀值，但仍按上下文替换以防误伤。
2. **同值多义（context-split）**：少数 hex 在同一文件里承担两种语义（已在各任务显式列出），**这些不可用 `replace_all`**，须逐处按上下文替换；其余唯一语义的值可 `replace_all`。
3. **验证零色**：每个组件文件改完跑 `rg -n --pcre2 '#[0-9a-fA-F]{3,8}\b|rgba?\(' <file>`，预期**无输出**（`color-mix(...)` 与 `var(...)` 不含 hex/`rgba(`，不会命中）。`theme.ts`/`TOKENS.md` 含 hex 属令牌真源，**不在零色范围**。
4. **stylelint**：`cd web; npx stylelint "<相对 src 的路径>"` 退出码 0。
5. **vite build 慢**：仅 Task 1（令牌 sanity）与 Task 6（终审）跑 `npx vite build`；中间任务不跑。
6. **提交**：`vue-tsc` 基线红可忽略（见 frontend-typecheck-baseline-red）。提交在 master，不推远程。
