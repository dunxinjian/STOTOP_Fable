# FIN科目去组织作用域（物理删 F组织ID 列）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 FinAccount / FinAccountBalance / FinAuxiliaryBalance 从组织作用域（IOrgScoped）改为账套作用域（IAccountSetScoped），移除冗余 IgnoreQueryFilters，并物理删除三张表的 F组织ID 列。

**Architecture:** 新增空标记接口 `IAccountSetScoped` 取代 `IOrgScoped`（后者触发 DbContext 的组织全局查询过滤器，前者不挂任何过滤器，故 DbContext 零改动）。删除三实体的 `FOrgId` 属性与配置映射，使 EF 模型不含该列；同步去掉因此变冗余的 17 处 `IgnoreQueryFilters`（保留 1 处对 FinVoucherEntry 的合法跨组织查询）；改写 909 条基线种子 INSERT 去掉该列，并新增 V5 迁移调用 `SeederHelper.DropColumnSafe` 从既存库删列。

**Tech Stack:** .NET / EF Core / SQL Server；xUnit 测试；自研 SchemaAutoSync + MigrationRunner 建库/迁移机制。

**设计依据：** [docs/superpowers/specs/2026-06-16-FIN科目去组织作用域-design.md](../specs/2026-06-16-FIN科目去组织作用域-design.md)
