# 角色三层「今天」主页骨架 Implementation Plan（绿色项③b）

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实施。`- [ ]` 复选框跟踪。

**Goal:** 新建登录后的「今天」主页 `/home`，按当前用户**角色自动呈现三张脸之一**（决策层经营脉搏 / 执行层待办流 / 轻量层"我要做什么"），并提供「视角」切换给身兼数职者。**规则版骨架、无 AI**：只拼装现成可复用组件 + 已有数据/计数 API，不造假数；尚无可靠数据源的经营指标（今日收入/毛利率/件量）以明确标注的"待接入"占位呈现（其真数据口径待定，属红色项）。

**Architecture:** 新建 `web/src/views/home/`：`useTodayRole.ts`（角色判定 + 视角覆盖）、`TodayHome.vue`（外壳：问候 + 视角切换 + 动态渲染层）、`DecisionLayer.vue` / `PowerLayer.vue` / `LightLayer.vue` 三层。`/home` 加为 `layoutRoute` 静态子路由，默认着陆由 `/workhub` 改为 `/home`（workhub 仍可访问，作为执行层"进入处理"的去处）。

**Tech Stack:** Vue 3 + Pinia + Ant Design Vue。复用件：`QualitySummaryCard`/`WorkHubRecentVisits`（零 props）、`StatCard`（`web/src/components/StatCard.vue`，按需 import）、`getWorkHubStats`/`getWorkItems`（`web/src/api/workhub.ts`）、`notificationStore.todoCount`、`permissionStore.taskCount`/`voucherPendingCount`/`hasAdminAccess`/`hasModuleAccess`、`userStore.roles`/`permissions`。组织取数靠请求头 `X-Org-Context` 自动注入。**无单测**，验证 `npm run build` + 预览。

## Task 2: 角色判定 composable

**Files:** Create `web/src/views/home/useTodayRole.ts`

- [ ] **Step 1:**
```ts
import { ref, computed } from 'vue'
import { useUserStore } from '@/stores/user'
import { usePermissionStore } from '@/stores/permission'

export type TodayRole = 'decision' | 'power' | 'light'
const OVERRIDE_KEY = 'today_home_role'

export const ROLE_LABELS: Record<TodayRole, string> = {
  decision: '经营视角',
  power: '处理视角',
  light: '快捷视角',
}

export function useTodayRole() {
  const userStore = useUserStore()
  const permissionStore = usePermissionStore()
  const override = ref<TodayRole | null>((localStorage.getItem(OVERRIDE_KEY) as TodayRole) || null)

  const detected = computed<TodayRole>(() => {
    const isBoss =
      userStore.roles.includes('admin') ||
      userStore.roles.includes('Admin') ||
      permissionStore.hasAdminAccess
    if (isBoss) return 'decision'
    const isPower =
      permissionStore.hasModuleAccess('finance', userStore.permissions) ||
      permissionStore.hasModuleAccess('cardflow', userStore.permissions) ||
      permissionStore.hasModuleAccess('quality', userStore.permissions)
    return isPower ? 'power' : 'light'
  })

  const role = computed<TodayRole>(() => override.value ?? detected.value)

  function setRole(r: TodayRole | null) {
    override.value = r
    if (r) localStorage.setItem(OVERRIDE_KEY, r)
    else localStorage.removeItem(OVERRIDE_KEY)
  }

  return { role, detected, override, setRole }
}
```
（`hasAdminAccess` 是 store computed，Pinia 解包后直接是 boolean；`hasModuleAccess` 是函数。若签名不符以实际为准。）
